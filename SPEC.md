# QR오더 사장님 어드민 — 개발 상세 명세서

본 문서는 소프트먼트가 운영하는 **QR오더 사장님 어드민** 웹 서비스의 화면·기능·데이터·동작을 개발팀이 그대로 구현할 수 있도록 정리한 1차 명세입니다. 인터랙티브 프로토타입(`index.html`)이 시각 기준이며, 본 문서가 **최종 확정 사양**입니다. 프로토타입과 본 문서가 충돌하면 본 문서가 우선합니다.

## 1. 공통 기준

### 1.1 영업일 정의

| 항목 | 정의 |
|---|---|
| 영업일 시작 | 03:00 KST |
| 영업일 종료 | 다음 날 02:59 KST |
| '오늘' | 현재 시각이 속한 영업일 |
| '어제' | 현재 영업일의 직전 영업일 |

매장 영업시간(예: 11:00 ~ 22:00)과 영업일(03:00 ~ 02:59)은 별개입니다. 영업일은 매출 집계·재고 차감의 기준이며, 매장 영업시간은 손님 주문 가능 시간을 나타냅니다.

### 1.2 타임존 / 로케일

- 타임존 : Asia/Seoul (UTC+9) 고정. 다국가는 1차 릴리즈 범위 외.
- 언어 : 한국어. 손님 화면 다국어는 `spec/02-menu.md` §6 번역 기능 참조.
- 날짜 표기 : `YYYY-MM-DD(요일)` 화면용, `YYYY-MM-DDTHH:mm:ss+09:00` 저장용.
- 시각 표기 : 24시간제 `HH:mm`.

### 1.3 통화 / 숫자

- 통화 : KRW 고정, 소수점 없음.
- 천 단위 구분자 : `,` (예: `1,234,500원`).
- 부가세 산식 : `vat = round(taxable / 11)` (일반과세 기준, 매장 마스터 `business_type=general`).
  - 간이과세 매장은 `vat = 0` 으로 표시하고 정산 시 별도 처리 (PG 정산 연동에서 조정).
  - 면세 매장은 1차 릴리즈 범위 외. 메뉴별 면세 토글은 §1.4 참조.
- 반올림 : 모든 통화 계산은 원 단위 `round half up`.

### 1.4 부가세·과세 처리

| 매장 사업자 유형 | 부가세 표시 | 정산 |
|---|---|---|
| 일반과세 (default) | 모든 매출 화면에 `vat`·`net` 컬럼 표시 | PG 정산은 `gross` 기준, 부가세 자체 신고 |
| 간이과세 | 모든 화면 `vat=0` 으로 노출 (UI 숨김은 아님) | 동일 |
| 면세 | 1차 범위 외 | — |

매장 마스터에 `business_type` 필드 추가(`general` / `simplified`). 가맹점 정보 화면(`spec/07-store.md`)에서 설정. 1차는 owner가 직접 설정하고 본사 승인은 정산 계좌 변경에만 적용.

### 1.5 환불 처리 규칙 (중요)

> 환불은 **발생일이 아닌 원거래(결제)일 매출에서 차감**됩니다. 세무·회계 표준 관행.

- 환불 트랜잭션은 `refund` 테이블에 별도 기록. 원거래 ID(`original_payment_id`)와 환불일시(`refunded_at`)를 함께 저장.
- 매출 집계 함수는 `refunds.original_paid_at` 의 영업일을 기준으로 차감합니다.
- 이미 정산이 완료된 일자의 매출이 사후 환불로 감소할 수 있습니다. 정산 시스템은 이 경우 **다음 정산 회차에서 차감 처리**합니다 (`spec/01-sales.md` §4 정산 참조).
- 부분 환불 시 결제건수는 1 유지, 환불건수는 1 증가 (부분 환불도 1건의 환불 이벤트로 카운트).

### 1.6 권한 (RBAC)

#### 1.6.1 역할

| 역할 | 정의 |
|---|---|
| `owner` | 사장님. 매장 마스터 데이터 변경·매출·정산·권한 관리 모두 가능 |
| `staff` | 직원. 운영 관련 기능만 가능. 매출 수치·정산·권한 관리·결제 관리 불가 |

`partial` (시간제 알바) 등의 추가 역할은 1차 범위 외. 차후 직원 권한 세분화로 확장.

#### 1.6.2 권한 매트릭스

| 영역 | owner | staff |
|---|---|---|
| 매출 한눈에 보기 | ✓ | ✗ |
| 매출 상세 보기 | ✓ | ✗ |
| 주문 내역 (상태 변경) | ✓ | ✓ |
| 주문 내역 (환불 처리) | ✓ | ✗ |
| 매출 정산 출금 | ✓ | ✗ |
| 메뉴판 (가격 수정) | ✓ | ✗ |
| 메뉴판 (사진·설명 수정, 카테고리 이동) | ✓ | ✓ |
| 옵션 그룹·세트·원산지·번역 | ✓ | ✓ |
| 오늘의 준비량 (수량 변경) | ✓ | ✓ |
| 오늘의 준비량 (오늘 품절 토글) | ✓ | ✓ |
| 주문 방식 변경 | ✓ | ✗ |
| 직원호출 메시지 관리 | ✓ | ✗ |
| 테이블 관리 | ✓ | ✓ (정렬·이름 변경만) |
| QR 관리 (재발급 신청) | ✓ | ✗ |

`✗` 영역에 접근 시 화면은 보여 주되 액션 버튼은 비활성 + 툴팁 "사장님 권한이 필요해요" 표시. 좌측 메뉴는 모두 노출하되 권한 없는 항목은 회색 + 자물쇠 아이콘.

### 1.7 동시성 / 데이터 정합성

| 영역 | 정책 |
|---|---|
| 재고 차감 (`MENU_STOCK.remaining`) | DB 트랜잭션 + `SELECT ... FOR UPDATE` 비관적 락. §`STATES.md` 참조 |
| 주문 상태 전이 | 낙관적 락 (`version` 컬럼). 충돌 시 최신 상태 재조회 후 재시도 |

| 사장님 / 직원 동시 작업 | 별도 잠금 없음. 변경은 변경 시각 기준 최신 우선 |

## 2. 비즈니스 규칙 모음 (전 화면 공통)

### 2.1 매출 계산식

```
gross    = sum(payment.amount)          -- 결제 승인 합계 (부가세 포함)
disc     = sum(discount.amount)         -- 쿠폰·해피아워 등 사장님 할인 합계
pt       = sum(point.amount)            -- 손님이 사용한 포인트
ref      = sum(refund.amount)           -- 환불 합계 (원거래일 기준)
actual   = gross - disc - pt - ref      -- 실매출
taxable  = actual                       -- 1차는 전체 과세 (면세 메뉴 분리 X)
vat      = round(taxable / 11)          -- 일반과세 기준
net      = actual - vat                 -- 순매출 (사장님 공급가액)
ocnt     = count(payment WHERE status='paid')
rcnt     = count(refund)                -- 부분환불 포함
avg      = actual / ocnt                -- 객단가
```

### 2.2 시간대 매출 데이터 출처

- 당일 데이터 : 결제 트랜잭션 raw 데이터를 시간 단위 `GROUP BY` (실시간 반영)
- 과거 데이터 : 영업일 마감 시점에 시간 단위로 사전 집계된 테이블에서 조회 (응답 빠름)
- 영업일 마감 처리는 영업일 종료(다음 날 02:59:59) 직후 배치로 수행

### 2.3 메뉴 효과적 상태 (`effectiveStatus`)

우선순위 (위에서 아래로):

```
1. manualSoldOut = true                       → 수동 품절
2. stock_item.remaining = 0                   → 자동 품절
3. menu.status = 'hidden'                     → 숨김
4. (else)                                     → 판매중
```

레거시 `status='soldout'` 값은 마이그레이션으로 `manualSoldOut=true` + `status='available'` 로 변환 후 enum에서 제거.

### 2.4 손님 화면 메뉴 노출 규칙

- 효과적 상태가 `숨김` : 손님 화면에 표시 안 함
- 효과적 상태가 `수동 품절` 또는 `자동 품절` : 손님 화면에 회색 + '품절' 라벨로 표시 (선택 불가)
- 효과적 상태가 `판매중` + 판매 시간(`schedule`) 조건 충족 : 선택 가능
- 판매 시간 조건 미충족 : 회색 + '판매 시간 아니에요' 라벨 (선택 불가)

### 2.5 영업일 마감 / 시작 자동 처리

| 데이터 | 자동 동작 |
|---|---|
| `MENU_STOCK.prepared` | **유지** (사장님 의도와 다른 변동 방지). 사장님이 매일 직접 갱신 |
| `MENU_STOCK.remaining` | 새 영업일 시작 시 `remaining = prepared` 로 리셋 |
| `MENU_STOCK.yesterdayPrep` / `yesterdaySold` | 새 영업일 시작 시 직전 영업일 값으로 갱신 |
| `MENUS.manualSoldOut` | 새 영업일 시작 시 **모두 자동 해제** (`'오늘 품절'` 라벨과 일치) |
| 영업일 시간대별 매출 raw | 영업일 마감 시 사전 집계 테이블에 적재 |

영업일 시작 시점(03:00) 사장님이 매장 어드민에 접속해 있는 경우, 화면을 부드럽게 갱신(전체 새로고침 없이 데이터만 재조회).

### 2.6 정렬 / 검색 / 페이징 공통

- 표 헤더 클릭 정렬 (오름차순 ↔ 내림차순 토글, 정렬 활성 컬럼에 화살표 표시)
- 검색은 부분 일치 (대소문자 무시, 한영 자동 변환 미지원)
- 페이지당 행 수 : 10 / 20 / 50 / 100 (기본 20)
- 페이지네이션 : 이전/다음 + 페이지 번호

### 2.7 모달 / 팝업 공통

- ESC 키로 닫기 가능
- 배경 클릭으로 닫기 가능 (단, 폼 입력 중 dirty 상태면 확인 prompt)
- 닫기 버튼은 우상단 `✕`
- 폼이 있는 모달의 dirty 상태 표시 + 변경사항 있을 때 닫기 시 확인

### 2.8 저장 / 변경 사항 추적

여러 화면(주문 방식 등)에서 "저장하지 않은 변경 사항이 있어요" 표시. 패턴:

- 화면 진입 시 상태를 `saved snapshot` 으로 저장
- 변경 발생 시 현재 상태 ≠ snapshot → dirty
- 다른 섹션 이동 시 dirty면 확인 prompt 노출 ("변경 사항을 버리고 이동할까요?")
- 저장 성공 시 snapshot 갱신
