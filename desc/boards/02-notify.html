🔔알림 센터 기본 notify
📍 핀 3개
1화면 개요 · 미확인 배지
기능·목적
  • 알림 센터 화면의 목적은 소프트먼트(운영사)와 소속 본사가 보내는 공지·안내를 한 곳에서 확인하는 것
  • 실시간 주문·재고·리뷰 알림은 각 기능 페이지에서 다루고, 이 화면에는 상위 조직 공지만 표시 (발신처 3종: 시스템 관리자/기업 관리자/브랜드 관리자)
  • 상단 헤더에 제목 공지·안내와 안내 문구, 우측에 확인이 필요한 공지 강조 칩을 배치해 사장님이 미확인 건을 바로 인지하게 함
동작·상태값
  • 미확인 건수는 사이드바 알림 센터 메뉴 우측 빨간 배지로 표시 (값: NOTIFICATIONS.filter(n=>!n.read).length, 0이면 배지 숨김)
  • 헤더 강조 칩은 🔔 N건 확인해 주세요 형태로 미확인 수를 노출, 클릭 시 state.notifyOnlyUnread 토글 → 활성 시 칩 색 반전 + 라벨 끝에 · 보는 중 추가
  • 홈 대시보드 "챙겨야 할 일" 카드에도 미확인 수가 합산 노출 (notifyUnread), 카드 클릭 시 setSection('notify')로 진입
  • 최초 진입 기본 상태: 발신처 필터 접힘(notifyFilterOpen=false), 전체 발신처(notifyFilter='all'), 미확인만 보기 꺼짐(notifyOnlyUnread=false)
데이터·연동
  • 공지 엔터티 NOTIFICATIONS[]: id, level(발신처), tag(분류), urgent(긴급 여부), title, body, issuer(표시용 발신 기관명), date, read(읽음 플래그)
  • 발신처 level 3종 — softment(시스템 관리자/운영사), corp(기업 관리자), brand(브랜드 관리자); 표시 메타는 NOTIFY_LEVELS(label·short·color·bg)
  • 본사/운영사 공지는 어드민 상단 배너 + 앱 푸시 동시 발송 — SPEC §1.8, 알림 센터는 그 수신 이력의 영구 보관함 역할
  • 미확인 카운트는 클라이언트가 !read 건수를 집계해 사이드바 배지·홈 카드·헤더 칩 세 곳에 동시 반영
예외처리·UX문구
  • 문구는 능동·긍정 톤 유지 — 미확인을 알릴 때 "미확인 N건"이 아닌 🔔 N건 확인해 주세요로 표현 (부정형 회피)
  • 안내 문구는 소프트먼트와 소속 본사에서 보내는 공지·안내를 한 곳에서 확인해 주세요.
  • 공지가 0건일 때는 빈 화면 대신 모두 확인했음을 긍정적으로 안내 (홈 카드 기준 모든 알림을 확인하셨어요)
  • staff·매니저 권한도 공지는 읽기 전용 — 발송·삭제 권한은 상위 조직에만 있고 사장님은 확인만 가능
2공지 목록 · 발신처 필터 · 모두 읽음
기능·목적
  • 공지 목록은 전체 공지를 한 목록으로 조회하고, 발신처별 필터와 미확인만 보기로 원하는 공지를 빠르게 좁히는 영역
  • 모두 읽음 처리 버튼으로 누적된 미확인을 한 번에 정리할 수 있게 함
  • 목록 헤더에 현재 노출 건수와 전체 건수를 함께 보여 필터 적용 상태를 인지하게 함 (공지 목록 N건 · 전체 M건 중)
동작·상태값
  • 기본 정렬: 미확인 + 긴급이 위로, 그다음 발행일시 최신순 (가중치 (!read?2:0)+(urgent?1:0) 내림차순, 동률 시 date 최신순)
  • 발신처 ▼ 버튼 클릭 시 필터 줄 펼침(notifyFilterOpen 토글), 칩 4종 — 전체 / 🟦 시스템 관리자 / 🟪 기업 관리자 / 🟩 브랜드 관리자, 각 칩에 건수 배지, 선택 시 notifyFilter 갱신 + 버튼에 활성 점(●) 표시
  • 모두 읽음 처리 클릭 시 markAllNotifyRead() 실행 → 전 공지 read=true 일괄 처리 후 재렌더(배지·카운트 즉시 갱신)
  • 안 읽음 공지는 강조 표시(제목 굵게 800, 우측 빨간 점), 읽은 공지는 opacity:.65로 흐리게 처리
  • 미확인만 보기(헤더 칩) + 발신처 필터는 중첩 적용되며, 행 클릭 시 상세로 진입
데이터·연동
  • 읽음 플래그 notification.read (boolean) — 목록 강조·정렬·미확인 집계의 단일 기준
  • 발신처 구분 sender/level 값으로 필터 (all|softment|corp|brand), 발신처별 건수 softCnt·corpCnt·brandCnt 집계
  • 필터 상태는 화면 로컬 상태 state.notifyFilter·state.notifyFilterOpen·state.notifyOnlyUnread로 보관 (구버전 'unread' 값은 notifyOnlyUnread=true로 자동 이관)
  • 발행일시 date는 목록에서 M/D HH:mm 축약 표기, 발신처 기관명 issuer와 함께 메타 줄에 노출
예외처리·UX문구
  • 새 공지를 모두 확인한 상태 안내: 능동·긍정으로 새로운 공지를 모두 확인하셨어요 (부정형 회피)
  • 필터·미확인 조건으로 결과가 0건일 때는 빈 상태 문구 다른 조건으로 한번 살펴봐 주세요. 노출 (없다고 단정하지 않음)
  • 모두 읽음 처리는 사이드바 배지·헤더 칩이 즉시 0으로 줄어드는 것으로 처리 결과를 피드백
  • 권한과 무관하게 읽음 처리·필터는 모든 사용자가 사용 가능 (조회·확인 전용 화면이라 별도 권한 제약 없음)
3개별 공지 항목
기능·목적
  • 개별 공지 항목을 클릭하면 상세 모달로 공지 본문 전체를 확인할 수 있음
  • 목록에서는 본문을 2줄로 요약(line-clamp:2) 노출하고, 상세에서 전체 본문과 문의처 안내를 제공
  • 긴급·점검 공지는 목록·상세에서 시각적으로 우선 인지되도록 강조
동작·상태값
  • 행 클릭 시 openNotifyDetail(id) 호출 → 해당 공지 read=true 처리 + 상세 모달 펼침 (열람 즉시 읽음 처리되어 미확인 카운트 감소)
  • 상세 모달 구성: 발신처 배지 + (긴급 시) 긴급 배지 + tag 배지, 제목, 발신처 · 발행일시 메타, 본문(white-space:pre-line), 문의 안내 카드, 닫기 버튼
  • 긴급(urgent) 공지는 상단 강조 — 정렬 가중치로 목록 최상단 배치 + 빨간 긴급 배지 노출 (예: 정기 점검·정책 변경)
  • 목록 행은 마우스 오버 시 배경 강조(--bg-2)되어 클릭 가능함을 표시
데이터·연동
  • 공지 본문 body, 발행일시 date(상세는 YYYY-MM-DD HH:mm 원본 표기), 중요도 urgent, 분류 tag(예: 점검·정책·약관·정산·신메뉴·이벤트·프로모션·안내)
  • 발신처별 문의처 매핑: softment→고객센터(1644-0000), brand→본사 매장운영팀, corp→본사 사무국
  • 발신처 표시 메타 NOTIFY_LEVELS[level](label·short·color·bg)로 배지 색·라벨 결정, 본문 첨부(단가표 등)는 본문 내 안내로 연동
  • 열람 시 갱신되는 read 플래그가 목록 강조·사이드바 배지·홈 카드 카운트에 즉시 반영
예외처리·UX문구
  • 긴급/점검 공지는 색만으로 의미를 전달하지 않고 빨간 색상 + 긴급 텍스트 배지·아이콘을 병행 — SPEC 08 §7.1(색만으로 의미 전달 금지, 아이콘/텍스트 라벨 병행)
  • 상세 모달 하단 안내 카드는 긍정 톤으로 문의를 유도: 💡 {발신처}에서 보낸 {분류}예요. 추가 문의는 {문의처}으로 부탁드려요.
  • 존재하지 않는 id로 상세 호출 시 모달을 열지 않고 무시(가드 if (!n) return;)해 오류 노출을 방지
  • 공지는 사장님이 수정·삭제할 수 없는 읽기 전용이므로 상세에는 확인용 닫기 액션만 제공
