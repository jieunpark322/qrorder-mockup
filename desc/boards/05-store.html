🏪매장 설정 매장 운영 store
📍 핀 6개
1상단 바 · 저장
기능·목적
  • 화면 제목 영역과 전역 저장 메커니즘. 매장 설정 화면(URL admin.qrorder.softment.co.kr/store)의 최상단 크럼브 '매장 운영 › 매장 설정' + 제목 h2 '매장 설정' + 부제 '매장 정보·운영 시간·휴무일을 관리해요.'를 표시한다.
  • 이 화면의 모든 패널(기본 정보·이미지·소개/편의정보·운영 시간·휴게 시간·휴무 일정)에서 일어난 변경을 한 번에 서버에 반영하는 진입점. 화면 내 입력은 메모리(STORE_INFO)에만 반영되므로 명시적 저장이 필요하다.
동작·상태값
  • 진입 시 STORE_INFO 전체를 deep-copy 해 snapshot으로 보관하고, 이후 현재 값과 비교해 dirty(변경 여부)를 추적한다(SPEC §2.8).
  • 어떤 필드라도 snapshot과 달라지면 상단/저장 버튼 영역에 '저장하지 않은 변경 사항이 있어요' 안내를 띄우고 저장 버튼을 활성화(markUnsavedUI() 호출 지점: 운영시간 select onchange, 행 추가/삭제, 요일 토글). 편의정보/이미지/소개글 변경도 동일하게 dirty로 잡도록 저장 로직에 포함한다.
  • [저장] 클릭: 전체 폼 검증 통과 시 매장 마스터 PATCH 후 success 토스트 '매장 설정을 저장했어요'(초록, 3초, SPEC 08-common §1.1) 노출 + snapshot 갱신 + dirty 해제. 검증 실패 시 저장 중단하고 첫 오류 필드로 스크롤·포커스.
  • 저장은 명시적 액션이므로 토스트로 결과 표시(08-common §1.3). 200ms 이하 응답은 스피너 생략, 처리 중 버튼 라벨 '처리 중...'(08-common §2).
데이터·연동
  • 저장 대상: 매장 마스터(store) — displayName, phone, intro, amenities[], amenitiesCustom[], 로고/이미지 자산, scheduleSlots[], hours[].breakS/breakE, holidays[].
  • business_type(general/simplified)도 매장 마스터 필드이며 1차에서는 owner가 설정(SPEC §1.4, §1.6.2). 정산 계좌 변경만 본사 승인 대상이고 나머지 정보 변경은 즉시 반영.
  • API 성격: 단일 매장 마스터 조회(GET)·부분 갱신(PATCH). 운영 시간·휴무는 손님 주문 화면과 영업일 집계의 기준 데이터로 연동된다.
예외처리·UX문구
  • staff 권한은 매장 마스터 변경 불가(SPEC §1.6.2). 화면은 보여 주되 입력/버튼을 비활성화하고 툴팁 '사장님 권한이 필요해요'를 표시(SPEC §1.6.2 / 111행).
  • dirty 상태에서 다른 화면으로 이동·새로고침 시도 시 이탈 확인 안내 '저장하지 않은 변경 사항이 있어요. 이동할까요?'를 띄워 실수 손실을 방지.
  • 저장 API 실패 시 error 토스트(빨강, 자동 닫힘 없음, 08-common §1.1) '저장에 실패했어요. 잠시 후 다시 시도해 주세요'를 표시하고 입력값은 유지한다.
  • 검증 미통과 항목이 있으면 저장을 막고 해당 패널의 인라인 오류(예: 운영시간 모순, 빈 요일)를 먼저 해소하도록 안내.
2기본 정보 (🔒 사업자등록증 기준)
기능·목적
  • 기본 정보 패널(제목 '기본 정보', 부제 '사업자등록증 기준 정보예요'). 매장명·손님용 표시명·대표자명·사업자등록번호·전화번호·주소를 표시·관리한다.
  • 🔒 잠금 항목은 가입 시 사업자등록증으로 시스템 관리자가 설정한 법적 정보로, 매출/세금 계산과 정산의 기준이 되므로 사장님이 임의 수정하지 못하게 막는다.
동작·상태값
  • 상단에 노랑 안내 박스: '🔒 매장 이름·대표자명·사업자등록번호·매장 주소는 가입 시 사업자등록증 정보로 시스템 관리자가 설정해, 여기서는 바꿀 수 없어요. 변경이 필요하면 고객센터로 알려 주세요.'
  • 잠금 필드(disabled): '매장 이름 🔒'(name), '대표자명 🔒'(owner), '사업자등록번호 🔒'(bizNo), '매장 주소 🔒'(addr) — 읽기 전용으로만 표시.
  • 수정 가능 필드: '손님이 보는 매장 이름'(displayName, oninput으로 즉시 반영, placeholder '손님 화면에 보일 이름', 보조문구 '손님 주문 화면에 이 이름이 보여요. 매장 이름과 다르게 자유롭게 정할 수 있어요.'), '매장 전화번호'(phone).
  • business_type(일반/간이과세) 설정 컨트롤을 이 패널에 둔다(SPEC §1.4). 기본값 general. 간이과세 선택 시 전 화면 vat=0로 노출(UI 숨김 아님, §1.4 64행).
데이터·연동
  • 기본값 예시: name='소프트커피 합정점', displayName='소프트커피 합정점', owner='홍길동', bizNo='123-45-67890', phone='02-1234-5678', addr='서울 마포구 양화로 45'.
  • 필드 타입/검증: displayName string 필수, 1~30자 권장. phone string, 전화번호 형식(숫자·하이픈) 검증. business_type enum(general/simplified) — 부가세 산식 vat=round(taxable/11)의 분기 기준(SPEC §1.3).
  • 연동: displayName·로고·소개·운영시간은 손님 주문 화면, business_type은 메뉴/주문/매출/정산 화면의 부가세 표기에 영향. 잠금 항목은 시스템 관리자(본사) 마스터에서만 변경.
예외처리·UX문구
  • 잠금 필드 수정 시도 시 안내 '변경이 필요하면 고객센터로 알려 주세요' 경로만 제공하고 직접 편집 불가.
  • displayName을 비우고 저장 시 인라인 오류 '손님에게 보일 매장 이름을 입력해 주세요'.
  • phone 형식 오류 시 '전화번호 형식을 확인해 주세요' 인라인 표시.
  • staff 권한은 본 패널 입력 비활성 + 툴팁 '사장님 권한이 필요해요'(SPEC §1.6.2).
3매장 소개 · 편의정보
기능·목적
  • 매장 소개·편의정보 패널(제목 '매장 소개 · 편의정보', 부제 '손님 주문 화면에 함께 보여요') + 로고/대표 이미지 관리. 손님이 주문 화면에서 보는 매장 소개글·편의정보 칩·이미지를 입력한다.
  • 매장의 특징(주차·와이파이 등)과 분위기를 손님에게 전달해 주문 경험을 높이는 콘텐츠 영역.
동작·상태값
  • 매장 로고(1장): 80x80 정사각 업로드 카드(클릭 시 업로드). 보조문구 'PNG·JPG / 정사각형 권장 / 최대 2MB'.
  • 매장 대표 이미지(최대 3장): addStoreImage()로 추가, 각 카드 ✕로 removeStoreImage(idx) 삭제. 3장 도달 시 '+ 이미지 추가' 카드 숨김. 보조문구 'PNG·JPG / 가로형 권장 / 장당 최대 5MB'. (주의: mockup의 extraImages<2 한계는 버그성 — 기획상 최대 3장으로 구현.)
  • 매장 소개글(textarea 3행, intro, oninput 즉시 반영, placeholder '예: 매일 직접 로스팅한 원두로 내린 커피와 갓 구운 베이커리를 즐길 수 있는 동네 카페예요.').
  • 편의정보 칩: 프리셋 14종(주차장·단체석·포장·무선 인터넷·남성/여성 화장실 구분·룸·국민지원금·지역화폐(지류형)·지역화폐(카드형)·제로페이·테라스·발렛파킹·반려동물 동반·유아용 의자)을 칩 클릭으로 선택/해제(toggleAmenity). 보조문구 '해당하는 항목을 눌러 선택할 수 있어요. 다시 누르면 해제돼요.'
  • 직접입력: 텍스트 입력 후 '+ 추가'(addCustomAmenity(), Enter 동일)로 amenitiesCustom에 칩 추가, 칩의 ✕로 removeCustomAmenity(i) 삭제. 보조문구 '목록에 없는 편의정보를 직접 적어 추가할 수 있어요.'
데이터·연동
  • 기본값: intro='매일 직접 로스팅한...', amenities=['주차장','무선 인터넷','포장'], amenitiesCustom=[], extraImages=0.
  • 필드: store.intro string(선택, 권장 0~200자), amenities string[](프리셋 라벨 부분집합), amenitiesCustom string[](자유 텍스트), 로고/이미지 자산 URL. 프리셋 라벨↔아이콘 매핑은 STORE_AMENITY_ICONS.
  • 연동: 소개글·편의정보·로고·대표 이미지는 모두 손님 주문 화면에 노출. 이미지는 업로드 시 형식(PNG/JPG)·용량(로고 2MB, 이미지 5MB) 검증.
예외처리·UX문구
  • 선택 입력 — 소개글·편의정보·이미지 모두 빈 값 허용. 손님 화면에서는 값이 있는 항목만 노출.
  • 직접입력에서 이미 선택/추가된 항목 재입력 시 info 토스트 'ℹ️ 이미 추가된 편의정보예요'를 띄우고 입력창만 비운다.
  • 빈 문자열로 '+ 추가' 시 추가하지 않고 입력창에 포커스만 유지.
  • 이미지 형식·용량 초과 업로드 시 '이미지는 PNG·JPG, 로고 2MB·대표 이미지 5MB 이하로 올릴 수 있어요' 안내. staff는 변경 비활성 + 툴팁 '사장님 권한이 필요해요'.
4매장 운영 시간
기능·목적
  • 매장 운영 시간 패널(제목 '매장 운영 시간'). 요일을 묶은 슬롯(알람 스타일)별로 오픈·마감·라스트오더를 설정해 손님이 주문 가능한 시간을 정의한다.
  • 손님 주문 화면의 영업 여부·라스트오더 마감 판단의 기준이 된다. 영업일(03:00 기준 매출 집계)과는 별개 개념(SPEC §1.1).
동작·상태값
  • [+ 행 추가](addScheduleSlot()): 빈 요일·open 09:00·close 22:00·lastOrder 빈값의 새 슬롯 추가. 여러 요일을 한 행에 묶어 관리. 보조문구 '요일을 선택하고 오픈·마감·라스트오더 시간을 입력하세요. 여러 요일을 한 행에 묶어서 관리할 수 있어요.'
  • 요일 칩(일·월·화·수·목·금·토 순): 클릭으로 슬롯에 요일 추가/해제(toggleSlotDay). 요일이 0개인 슬롯은 노란 강조 + 경고 '⚠️ 적용할 요일을 하나 이상 선택해 주세요.'
  • 시간 select(10분 단위 timeOptions10): 오픈·마감(필수, 기본 09:00/22:00), 라스트오더(빈값 허용 = '선택 안 함'). 변경 시 updateSlotTime + markUnsavedUI()로 dirty 처리.
  • [✕ 삭제](removeScheduleSlot(id)): 슬롯 2개 이상일 때만 노출. 마지막 1개 삭제 시도 시 차단(아래 예외).
  • 기본 슬롯: 슬롯1 월~금 09:00~22:00 라스트오더 21:30, 슬롯2 토·일 10:00~22:00 라스트오더 20:30.
데이터·연동
  • 필드: scheduleSlots[{id:number, days:string[], open:'HH:mm', close:'HH:mm', lastOrder:'HH:mm'|''}]. id는 기존 최대값+1로 부여.
  • 검증: open·close 형식 HH:mm, close>open(마감<오픈 모순 차단), lastOrder<=close 권장. 슬롯 간 같은 요일 중복 배정 금지. (참고로 기존 hours[].open/close 배열은 요일별 표시·휴게 기준으로 공존.)
  • 연동: 손님 주문 화면 영업 판단·라스트오더 안내에 사용. 영업시간 ≠ 영업일 — 매출/재고 집계는 03:00 시작 영업일 기준(SPEC §1.1).
예외처리·UX문구
  • 요일 미선택 슬롯은 인라인 경고 '⚠️ 적용할 요일을 하나 이상 선택해 주세요.' 표시 + 저장 차단.
  • 마지막 슬롯 삭제 시도 시 안내 '운영 시간은 최소 1개 필요해요.'(removeScheduleSlot alert) 후 삭제 취소.
  • 마감이 오픈보다 빠르거나 같은 모순 입력 시 인라인 오류 '마감 시간은 오픈 시간보다 늦어야 해요'.
  • 두 슬롯에 같은 요일이 중복되면 '같은 요일이 여러 행에 들어 있어요. 한 곳만 남겨 주세요' 안내. staff는 비활성 + 툴팁 '사장님 권한이 필요해요'.
5매장 휴게 시간 (브레이크)
기능·목적
  • 매장 휴게 시간(브레이크) 패널(제목 '매장 휴게 시간', 부제 '브레이크타임이 없는 요일은 비워두세요'). 요일별 휴게 시작·종료 시간을 설정한다.
  • 점심 마감 등 영업 중 잠시 주문을 받지 않는 시간대를 손님 화면에 반영하기 위한 영역.
동작·상태값
  • 요일 7행(월·화·수·목·금·토·일) 표 — 컬럼: 요일 · 휴게 시작 · ~ · 휴게 종료 · (요약). 헤더 보조 '비워두면 해당 요일 휴게 없음'.
  • 휴게 시작/종료는 10분 단위 select(updateBreak(i,'breakS'|'breakE',value), 빈값 허용). 둘 다 채우면 요약 칸에 'HH:mm ~ HH:mm', 아니면 '휴게 없음' 표시.
  • 브레이크 없는 요일은 두 칸을 비워두면 됨. 변경 시 dirty 처리되어 저장 대상에 포함.
  • 기본값: 월~목 15:00~17:00 브레이크, 금·토·일 브레이크 없음.
데이터·연동
  • 필드: hours[i].breakS / hours[i].breakE ('HH:mm'|''). 인덱스 i는 요일 순서(월=0...일=6) 고정.
  • 검증: 한쪽만 입력 불가(시작·종료 동시 입력 필요), breakE>breakS, 브레이크는 해당 요일 운영 시간(open~close) 범위 안이어야 함.
  • 연동: 손님 주문 화면의 브레이크 안내(대시보드 '오늘' 카드에도 '브레이크 HH:mm~HH:mm' 노출, index 2159행)에 사용.
예외처리·UX문구
  • 시작·종료 중 한쪽만 입력 시 인라인 오류 '휴게 시작과 종료를 모두 선택해 주세요'.
  • 종료가 시작보다 빠르면 '휴게 종료는 시작보다 늦어야 해요'.
  • 브레이크 중 손님 주문 처리 정책: 브레이크 시간대에는 손님 주문 화면에서 주문 버튼을 비활성화하고 '지금은 휴게 시간이에요. 잠시 후 다시 주문할 수 있어요'를 안내(설계 기준; 직원 호출은 별도 정책).
  • staff는 입력 비활성 + 툴팁 '사장님 권한이 필요해요'.
6공휴일 · 휴무 일정
기능·목적
  • 공휴일·휴무 일정 패널(제목 '공휴일 · 휴무 일정', 부제에 건수 'N건'). 특정 날짜 또는 사유 기반 휴무를 등록해 해당일 손님 주문을 차단한다.
  • 정기 운영시간과 별개로 임시·기념일 휴무를 관리하기 위한 영역.
동작·상태값
  • [+ 휴무일 추가](openHolidayModal()): 모달(제목 '휴무일 추가', 부제 '날짜를 지정하거나, 기념일·사유를 직접 입력할 수 있어요')에 두 탭 — '📅 날짜 지정' / '✏️ 수기 입력'(switchHdTab).
  • 날짜 지정 탭: 날짜(필수, 기본 오늘) + 사유(선택, placeholder '예: 성탄절, 어린이날'). 추가 시 'YYYY-MM-DD (사유)' 또는 'YYYY-MM-DD' 형태로 holidays에 push.
  • 수기 입력 탭: 기념일·사유(필수, placeholder '예: 추석 연휴, 개업 기념일, 대표자 개인 일정'). 보조문구 '날짜 없이 사유만 기록할 때 사용해요. 목록에 📌로 표시돼요.'
  • 모달 하단 [취소]/[추가](addHolidayFromModal()). 추가 성공 시 success 토스트 '📅 휴무일을 추가했어요'. 목록 각 항목은 날짜형이면 📅, 수기형이면 📌 아이콘 + [삭제](splice 후 재렌더).
데이터·연동
  • 필드: holidays: string[]. 날짜형 'YYYY-MM-DD (사유)' / 'YYYY-MM-DD', 수기형 자유 텍스트. 정규식 /^\d{4}-/로 날짜형 판별해 아이콘 분기.
  • 기본값: ['2026-01-01 (신정)','2026-05-05 (어린이날)']. 패널 제목 건수는 holidays.length로 바인딩.
  • 연동: 등록된 날짜형 휴무일에는 손님 주문 화면에서 주문을 차단. 정렬은 날짜 오름차순 권장(SPEC §2.6).
예외처리·UX문구
  • 빈 상태(0건): '[+ 휴무일 추가]로 첫 휴무일을 등록할 수 있어요.'(가능형, 부정 표현 회피, 08-common §3.2).
  • 날짜 미선택 추가 시 '날짜를 선택해 주세요.', 수기 사유 미입력 시 '기념일·사유를 입력해 주세요.' 안내 후 추가 중단.
  • 경계값: 과거 날짜는 의미가 없으므로 오늘 이전 날짜 입력 시 '지난 날짜는 휴무로 등록할 수 없어요. 오늘 이후로 선택해 주세요' 안내(과거 날짜 입력 제한). 동일 날짜 중복 등록 시 'ℹ️ 이미 등록된 휴무일이에요'.
  • staff는 추가·삭제 버튼 비활성 + 툴팁 '사장님 권한이 필요해요'.
