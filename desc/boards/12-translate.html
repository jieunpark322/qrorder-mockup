🌐번역 메뉴 관리 translate
📍 핀 4개
1언어 선택 · 개요
기능·목적
  • 손님이 보는 메뉴명·설명을 4개 언어(영어 en / 번체 zhTw / 간체 zhCn / 일본어 ja)로 입력·관리하는 화면의 언어 전환 탭과 전체 개요 영역
  • 선택한 언어 1개를 '현재 편집 언어'로 잡아 입력표 전체를 그 언어 기준으로 바인딩(한국어→선택 언어 1:1 편집)
  • 각 언어별 번역 완성도를 진행률(%)·n/총개수로 한눈에 보여 어느 언어가 덜 채워졌는지 파악하게 함
  • 이 데이터는 손님 QR 화면의 언어 전환 기능에 그대로 연결됨(SPEC §6, spec/02-menu.md §6)
동작·상태값
  • 언어 탭 4개: 클릭 시 state.trLang 변경 후 renderApp()로 표 재렌더. 기본값 'en'(영어). 활성 탭은 파란 강조(.t.on)
  • 각 탭 라벨 우측에 해당 언어 입력 완료 수 / 총 개수 표시(예: 3/9). 카운트는 현재 대상(메뉴/옵션/세트) 항목 중 tr[langCode]가 비어있지 않은 개수
  • 진행률 카드: 큰 퍼센트(pct=round(filled/total*100)) + 진행바(파란 채움) + '{flag} {label} 번역 진행률 — {대상종류} 기준' 캡션. 대상·언어 전환 시 즉시 재계산
  • 완료 요약 문구는 능동·긍정: 미입력이 있으면 '{filled}/{total}개 완료 · 미입력 N개'(앰버), 모두 채우면 '모두 완료'(초록)
  • 헤더 우측 pill에 현재 선택 언어를 표기('{flag} {label}'), 패널 제목 우측에 '한국어 → {flag} {label}' 방향 표시
  • 탭/대상/언어 전환은 저장 없이 즉시 반영(모든 입력은 입력 즉시 메모리 반영, 별도 저장 버튼 없음 — 실서비스는 자동저장 또는 명시 저장 중 택1, §5.4 참고)
데이터·연동
  • 언어 마스터 TR_LANGS = [{code:'en',label:'English',flag:🇺🇸}, {code:'zhTw',label:'繁體中文'}, {code:'zhCn',label:'簡体中文'}, {code:'ja',label:'日本語'}] — 4개 고정, 1차 릴리즈 범위(SPEC §1.1: 서비스 자체는 한국어, 손님 다국어만 이 4종)
  • 항목별 번역 필드: item.tr[langCode](메뉴명 번역), item.trDesc[langCode](설명 번역). 둘 다 trEnsure()로 4개 언어 키를 빈 문자열 보장
  • 확정 데이터 모델은 Translation 행 단위(DATA_MODEL §2.11): {store_id, entity_type(menu|category|option_group|option_item), entity_id, locale(en/ja/zh-CN 등), field(name|description), value(≤200자)}, UQ=(entity_type,entity_id,locale,field). 프로토타입의 tr/trDesc 맵은 이 행 집합으로 정규화 저장
  • 진행률 계산식: filled = 현재 대상 항목 중 tr[lang] 비어있지 않은 수, pct = total>0 ? round(filled/total*100) : 0 (total=0이면 0%)
  • 로케일 코드 매핑 주의: 화면 내부 코드 zhTw/zhCn ↔ 표준 zh-TW/zh-CN, en/ja는 동일 — 저장 시 표준 BCP-47로 정규화 권장
예외처리·UX문구
  • 권한: 번역은 owner·staff 모두 편집 가능(SPEC §1.6.2 '옵션 그룹·세트·원산지·번역' = owner ✓ / staff ✓). 별도 비활성 처리 없음
  • total=0(해당 대상에 항목이 하나도 없음)일 때 진행률 0%·진행바 빈 상태로 표기하고 표는 빈 상태 문구 노출(핀 2 참조)
  • 안내 보조문구는 가능형 유지: '손님이 보는 메뉴명·설명을 4개 언어로 입력해요. 사전 기반 자동 번역도 가능해요.'
  • 언어 추가/삭제는 1차 범위 외(4종 고정). 다국가·추가 로케일은 후속 — 모델은 locale 가변이므로 확장 가능
2번역 입력 표
기능·목적
  • 선택한 언어 기준으로 한국어 원문 → 번역을 입력하는 표 전체 영역과, 그 위의 대상 전환·검색·필터·일괄 자동번역 컨트롤
  • 번역 작업 범위(메뉴/옵션 그룹/세트 메뉴)를 좁히고, 미입력 항목만 빠르게 채워넣도록 돕는 작업 도구 모음
  • 표는 상품코드·원문(메뉴명+설명)·번역 입력칸·행 액션으로 구성되어 항목별 번역을 한 화면에서 처리
동작·상태값
  • 대상 chip 3종: '🍽️ 메뉴'(기본, state.trTarget='menu') / '🧩 옵션 그룹'('option') / '🎁 세트 메뉴'('set'). 클릭 시 표 데이터 소스를 MENUS/OPT_GROUPS/SETS로 교체하고 우측에 항목 수 표시
  • 검색 입력 '원문(한국어)으로 찾기': state.trSearch에 바인딩, 원문 name에 대소문자 무시 부분일치 필터. onchange(엔터/포커스아웃)에 재렌더
  • 토글 '미입력만 보기'(state.trOnlyEmpty, 기본 OFF): ON이면 현재 언어 tr[lang]가 빈 행만 노출. 검색과 AND 결합
  • 버튼 '⚡ 미입력 전체 자동 번역'(autoTranslateAll): 현재 대상의 미입력 항목만 사전 기반 자동번역. 이미 입력된 값은 건드리지 않음(덮어쓰기 없음). 사전에 있으면 tr/trDesc 채우고, 없으면 건너뜀
  • 표 헤더 4컬럼: '상품코드'(90px) · '원문 (한국어)'(35%) · '{flag} {label} 번역' · 행 액션(140px, 무라벨)
  • 정렬은 항목 원본 순서(메뉴판 노출 순서) 유지 — 별도 정렬 UI 없음. 검색·필터는 표시 행만 줄임(SPEC §2.6 검색·필터 준용)
데이터·연동
  • 대상별 소스: menu→MENUS(kind='메뉴'), option→OPT_GROUPS('옵션 그룹'), set→SETS('세트 메뉴'). 진행률·카운트도 이 대상 기준으로 분리 계산
  • 필터 파이프라인: rows = items.filter(name 부분일치).filter(미입력만 보기 ? !tr[lang] : true)
  • 자동번역 사전 TR_DICT: 한글 단어/구문 → {en,zhTw,zhCn,ja} 매핑(카페 메뉴 위주). trAuto(text,lang)는 ① 완전일치 우선 ② 없으면 부분 문자열 치환 ③ 매칭 0건이면 빈 문자열 반환
  • autoTranslateAll 결과 누계 cnt(번역됨)·skipped(사전 미등록)를 집계하여 결과 안내에 사용
  • 검색·토글 상태는 화면 로컬 상태(trSearch/trOnlyEmpty)이며 저장 대상 아님. 번역 값만 Translation으로 영속화
예외처리·UX문구
  • 빈 상태 2종(표 본문 colspan=4): 대상 항목이 0개면 '먼저 {대상종류}을 등록해 주세요.'(메뉴 화면으로 유도), 검색·필터로 결과가 0개면 '다른 조건으로 다시 찾아봐 주세요.' (§3.2 능동·긍정 톤)
  • 일괄 자동번역 결과는 안내로 노출: '자동 번역 결과 · 번역됨: {cnt}개 · 사전 미등록 (수동 입력 필요): {skipped}개' (프로토타입 alert → 실서비스 success/info 토스트 §1.1, 미등록 다수 시 직접 입력 유도)
  • 검색은 원문(한국어)만 대상(번역값으로는 검색 안 됨) — placeholder로 명시. 번역값 검색이 필요하면 후속 범위
  • 미입력만 보기 ON 상태에서 한 행을 채우면 다음 렌더에서 그 행이 목록에서 사라짐(완료 처리로 자연 감소) — 의도된 동작
  • 200ms 이하로 끝나는 자동번역은 로딩 표시 생략(§2.2). 실서비스 AI API 연동 시 인라인 스피너('처리 중...')와 5초/30초 타임아웃 규칙 적용
3메뉴별 번역 행
기능·목적
  • 메뉴(또는 옵션 그룹/세트) 한 항목당 한 행으로, 해당 항목의 메뉴명·설명을 현재 언어로 번역 입력·수정하는 단위 영역
  • 행 좌측에 원문 컨텍스트(상품코드·이모지·원문명·메타)를 함께 보여 어떤 상품을 번역 중인지 즉시 식별
  • 행 단위 자동번역과 행 단위 비우기로 항목별 빠른 편집 지원
동작·상태값
  • 번역 입력칸(메뉴명): oninput 시 setTranslation(target,id,'name',lang,value) → item.tr[lang] 즉시 갱신. placeholder는 자동번역 후보(trAuto(원문))가 있으면 그 값, 없으면 '번역 입력'
  • 설명 입력칸(2번째): 원문 desc가 있는 항목에만 노출. setTranslation(...,'desc',...)로 item.trDesc[lang] 갱신. placeholder는 자동번역 후보 또는 '설명 번역(선택)'
  • 버튼 '⚡ 자동'(autoTranslateOne): 그 행의 원문명/설명을 사전 기반 번역해 tr/trDesc[lang]에 채움(있으면 덮어씀). 행 단위는 명시 액션이므로 기존 값도 갱신
  • 버튼 '✕'(빨강): 현재 언어의 그 행 tr[lang]·trDesc[lang]를 빈 문자열로 비우고 재렌더. 번역값이 있을 때만 노출
  • 행 메타 표시 규칙: 메뉴='{카테고리|미분류} · {가격}원', 옵션그룹='{필수|선택} · 항목 N개', 세트='구성품 N개 · {가격}원'. 메뉴 행은 좌측에 대표 이모지(img, 기본 🍽️)
  • 입력은 즉시 메모리 반영(자동저장 성격). 표 재렌더 없이 타이핑되며 진행률/탭 카운트는 다음 렌더 때 갱신
데이터·연동
  • 행 키: item.id로 MENUS/OPT_GROUPS/SETS에서 항목 조회. 표시 상품코드 item.code(영문1자+숫자5 = 6자, 매장 내 유니크, M=메뉴/O=옵션/S=세트 — spec/02-menu §1.5.1·§8)
  • 원문 바인딩: origName=item.name, origDesc=item.desc. 번역 바인딩: trName=tr[lang], trDesc=trDesc[lang]
  • 검증: value 길이 ≤200자(Translation.value), 메뉴명 번역도 원문과 동일하게 1~30자 권장. HTML 주입 방지 위해 따옴표 이스케이프("→&quot;) 후 바인딩
  • 연동: 입력값은 손님 QR 화면(언어 전환)에서 해당 상품의 표시명/설명으로 노출. entity_type는 대상별로 menu/option_group(+option_item)/세트로 매핑(DATA_MODEL §2.11)
  • autoTranslateOne은 trAuto 결과가 빈 문자열이면 값을 쓰지 않고 안내만 표시
예외처리·UX문구
  • 행 자동번역에서 사전에 없으면: '사전에 없는 단어예요. 직접 입력해 주세요.' 안내(가능형·능동 톤). 값은 변경하지 않음
  • 번역 미입력 행은 손님 화면에서 원문(한국어) fallback 노출(spec/02-menu §6) — 즉 빈 칸이 손님에게 빈 값으로 보이지 않음
  • 설명 원문이 없는 항목은 설명 입력칸 자체를 숨김(불필요 입력 방지)
  • 입력 검증 실패(200자 초과 등) 시 §5.2: 필드 테두리 빨강 + 아래 빨간 메시지, 토스트 미사용. 경계값은 자동 절단이 아닌 입력 거부 권장
  • 권한 부족 상황은 발생하지 않음(번역은 owner/staff 공통). 단 가격은 표 메타로 '표시만' 하며 이 화면에서 수정 불가(가격 수정은 메뉴 화면·owner 전용 §1.6.2)
4자동 번역 안내
기능·목적
  • 표 하단 안내 배너로, '⚡ 자동 번역'이 어떤 방식으로 동작하는지(사전 기반)와 한계를 사장님에게 설명
  • 사전에 없는 단어는 직접 입력해야 정확하다는 점을 알려 자동번역 결과를 맹신하지 않도록 유도
  • 실서비스 고도화 방향(AI 번역 API 연동)을 개발팀에 명시하는 제품 노트 성격 포함
동작·상태값
  • 정적 정보 배너(.banner-info), 클릭·토글 등 인터랙션 없음. 표 아래 항상 노출
  • 자동번역 동작의 기준 정보를 제공(현재는 내장 TR_DICT 카페 메뉴 사전). 핀 2·3의 ⚡ 버튼 동작과 일관
  • 실서비스 전환 시 본 배너 문구를 'AI 번역으로 초안을 만들고 검수해 주세요' 류로 업데이트하도록 가이드
데이터·연동
  • 참조 데이터: 자동번역 사전 TR_DICT(한글→4개 언어). 완전일치 우선, 부분 치환 보조(trAuto)
  • 별도 저장·연동 필드 없음(순수 안내 텍스트)
  • 실서비스 권장: 외부 번역 API(예: 기계번역) 결과를 초안으로 Translation.value에 채우고 owner 검수 플로우 추가
예외처리·UX문구
  • 노출 문구 그대로: '💡 ⚡ 자동 번역은 카페 메뉴 사전을 기반으로 동작해요. 사전에 없는 단어는 직접 입력해야 정확해요. (실서비스에서는 AI 번역 API 연동을 권장)'
  • 톤: 능동·긍정·가능형 유지, 경고·공포 표현 금지(§3.2)
  • 배너는 빈 상태·에러와 무관하게 항상 표시(표가 비어 있어도 노출되어 다음 행동을 안내)
