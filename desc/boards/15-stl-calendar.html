🗓️정산 달력 정산 관리 stl-calendar
📍 핀 4개
1개요 (정산 주기 D+3)
기능·목적
  • 화면 최상단 크럼브·타이틀·서브카피·이동 버튼 영역. 크럼브 정산 관리 › 정산 달력, 타이틀 정산 달력, 서브 정산일을 한눈에 · 정산 주기 D+3 영업일으로 이 화면이 '정산금 입금일을 달력으로 보는 곳'임을 알린다.
  • 사장님이 언제 얼마가 입금되는지를 직관적으로 파악하고, 더 자세한 회차별 표가 필요하면 정산 내역으로 넘어가도록 동선을 제공하는 진입 헤더.
  • 정산 주기 규칙(결제일 + 3영업일 = 정산일)을 한 줄 카피로 상시 노출해 입금 예측의 기준을 명시한다.
동작·상태값
  • 우측 🔎 정산 내역 보기 버튼(btn-ghost) 클릭 → setSection('stl-list')로 정산 내역 화면 이동. 토글/상태 없는 단순 네비게이션.
  • 서브카피·타이틀은 정적 텍스트로, 월 이동(핀2)이나 선택(핀3)에 영향받지 않고 항상 동일하게 노출.
  • D+3는 달력(역일)이 아니라 영업일 기준이며 주말·공휴일은 카운트에서 제외(addBizDays가 토·일·isHoliday 스킵). SPEC §1.1 영업일(03:00~익일 02:59) 정의를 따른다.
데이터·연동
  • 화면 라우트 admin.qrorder.softment.co.kr/settlements/calendar. 데이터 소스는 정산 회차 컬렉션 SETTLEMENTS(파생 집계).
  • 정산일 산출식: settleISO = addBizDays(approvalDate, 3) — base 결제일에서 영업일 3일 가산. 계산식 근거 SPEC §1.1.
  • 연동: 정산 내역(stl-list), 회차 클릭 시 정산 상세(stl-detail). API 성격은 월 단위 정산 스케줄 조회(GET, read-only).
  • 권한 플래그: 정산 영역은 role=owner 전용. staff는 매출·정산 접근 불가(SPEC §1.6.2).
예외처리·UX문구
  • 권한 부족(staff 접근): 정산 메뉴 자체를 미노출하거나 진입 차단. 가능형 안내 "정산 정보는 사장님 계정에서 확인할 수 있어요."
  • 로딩 중에는 헤더는 즉시 그리고 달력 영역만 스켈레톤 처리(spec/08-common 로딩 규칙).
  • D+3 계산에서 연휴가 길면 정산일이 크게 밀릴 수 있음 — 카피는 항상 '영업일' 기준임을 명시해 오해를 방지.
2월 이동 · 범례
기능·목적
  • 월 이동 네비게이션 + 상태 범례 패널. 좌측은 ‹ / › 화살표와 YYYY년 M월 라벨, 우측은 정산 상태 색상 범례(입금 완료 / 곧 들어와요).
  • 사장님이 과거·미래 달의 정산 일정을 탐색하고, 달력 셀의 색이 무엇을 뜻하는지 즉시 이해하도록 돕는 컨트롤·범례 영역.
동작·상태값
  • ‹ 전월 버튼(cal-nav, aria-label '전월') → setCalendarMonth(-1), › 익월 → setCalendarMonth(1). 보고 있는 월 상태 state.calendarMonth(기본 '2026-05')를 갱신 후 재렌더.
  • 현재 월(2026-05)이 아닐 때만 오늘로 버튼(btn-ghost) 노출 → setCalendarMonth('today')로 기준월을 '2026-05'로 복귀. 현재 월에서는 숨김.
  • 범례: cal-legend done=입금 완료(정산완료), cal-legend pending=곧 들어와요(정산대기), cal-legend hold=확인 중(보류/held, STATES §8.2). 색은 의미와 1:1.
  • 월 라벨은 min-width:140px 중앙 정렬로 자릿수 변동에도 화살표 위치 고정.
데이터·연동
  • 상태 변수 state.calendarMonth(형식 'YYYY-MM'). setCalendarMonth는 new Date(y, m-1+delta, 1)로 연·월 경계(12월↔1월) 자동 처리.
  • isCurrentMonth = (monthKey === '2026-05')로 '오늘로' 버튼 노출 여부 결정. 운영 시 '오늘' 기준은 SPEC §1.1 영업일 기준 현재월로 대체.
  • 상태 매핑: statusPill = (status==='정산완료') ? 'done' : 'pending'. 정산완료/대기 판정은 settleISO ≤ todayISO 문자열 비교(타임존 버그 방지).
예외처리·UX문구
  • 색만으로 의미 전달 금지 — 각 셀에 텍스트 태그(입금 완료/곧 들어와요)를 병행하고 범례에도 라벨을 붙여 색각 이상 사용자 대응(접근성, spec/08-common).
  • 화살표로 무한히 과거/미래 이동 가능하나, 데이터 없는 달은 핀4의 빈 상태가 처리.
  • 전월/익월 버튼은 aria-label 제공으로 스크린리더 접근성 확보.
3정산 달력
기능·목적
  • 월간 정산 달력 그리드 본문. 요일 헤더(일~토)와 7열 날짜 셀로 구성되며, 정산이 발생한 날짜 셀에 입금(예정)액·상태 태그·해당 결제일을 표시한다.
  • 날짜별로 '그날 통장에 들어오는 정산금'을 한눈에 보여주고, 클릭으로 그날 회차 상세까지 바로 진입하게 하는 핵심 영역.
동작·상태값
  • 셀 렌더: 정산 있는 날만 cal-amt(금액 ${fmt(totalSettle)}원) + cal-tag(완료=입금 완료/대기=곧 들어와요) + cal-sub(결제일 M/D(요일)) 표기. 빈 날·월 밖 칸은 비워둠(cal-cell off).
  • 금액이 있는 날짜 클릭 → selectStlDate(iso): 해당 일자 정산 회차 상세(stl-detail)로 즉시 이동(중간 정산내역 단계 생략). 정산 없는 칸은 onclick 미부착(클릭 불가).
  • 오늘(2026-05-15)은 cal-day today 강조, 선택일(state.calendarSelectedDate, 기본 2026-05-15)은 cal-cell selected 강조.
  • selectStlDate 매칭 우선순위: ① settleISO===iso 직접 회차 → ② 같은 날 결제(approvalDays)에 속한 회차 → ③ 정산일이 가장 가까운 회차. 주말·공휴일 클릭 시 가장 가까운 회차로 진입.
데이터·연동
  • 인덱스 byDate[settleISO]=settlement로 정산일별 1셀 매핑. 첫 요일 new Date(year,month-1,1).getDay(), 말일 new Date(year,month,0).getDate()로 7배수 그리드 생성.
  • 셀 표시 필드: totalSettle(받으실/입금 금액, int), status(정산완료|정산대기), approvalDate·approvalDow(결제일·요일).
  • 계산식: totalSettle = (gross − refund − feeTotal) + couponPlatformAmt. 수수료 feeTotal = feeBase + round(feeBase×0.1), 부가세는 round 규칙(SPEC §1.3). 사장님(merchant) 발행 쿠폰은 정산 제외.
  • 같은 정산일에 여러 영업일 매출이 묶이면(주말·공휴일 이월) 그룹 합산되어 한 셀에 표시. 회차 ID STL-2026MMDD-MMDD.
  • 환불 사후 차감: 이미 정산 완료된 일자의 매출이 나중 환불로 줄면 발생 회차가 아니라 다음 정산 회차에서 차감(SPEC §1.5, §1.5 75행). 결제 취소 여부는 선/후불이 아니라 Payment.status=paid(환불 발생=결제 취소)로 판정.
예외처리·UX문구
  • 미래 달: 오늘 이후 결제는 아직 정산 대상이 아니라 회차 미생성 — 미래 정산 예정일은 곧 들어와요(pending)로만 표시.
  • 정산 없는 날 클릭 무반응(커서 기본). 정산 있는 셀만 cursor:pointer.
  • 환불 사후 차감으로 특정 회차 금액이 음수에 근접/감소할 수 있음 — 상세에서 차감 사유를 안내(SPEC §1.5).
  • 금액은 tabular-nums·천단위 콤마(fmt)로 정렬성 확보, 셀이 좁아도 잘리지 않게 표기.
4안내문
기능·목적
  • 달력 하단 안내 카드(info-card). 달력 사용법(날짜 클릭 = 상세 보기)을 능동·친근한 톤으로 알리고, 데이터가 없을 땐 빈 상태 메시지를 대신 보여주는 보조 영역.
동작·상태값
  • 데이터 있는 달(hasData=true): 💡 금액이 적힌 날짜를 누르면, 그날 받은 정산을 자세히 볼 수 있어요. 가능형 안내 노출.
  • 데이터 없는 달(hasData=false): 🗓️ 아이콘 + ${year}년 ${month}월 정산 내역이 없어요 + 보조 좌우 화살표로 다른 달을 확인할 수 있어요로 전환.
  • 클릭·토글 동작 없는 정적 안내. 핀2의 월 이동에 따라 hasData가 바뀌면 문구가 자동 전환.
데이터·연동
  • hasData = SETTLEMENTS.some(s => s.settleISO.startsWith(monthKey))로 분기. 별도 API 호출 없이 핀3과 동일 데이터셋 재사용.
  • 동적 바인딩 값은 year·month(보고 있는 월)뿐. 그 외는 고정 카피.
예외처리·UX문구
  • 빈 상태는 부정형('없어요'만 강조) 대신 다음 행동을 제시하는 가능형 보조문구(좌우 화살표로 다른 달을 확인할 수 있어요)를 병기 — 톤앤매너 능동·긍정 규칙 준수.
  • 빈 상태 카드 자체가 에러가 아닌 정상 상태이므로 에러 토스트를 띄우지 않음.
  • 안내 문구는 강조용 이모지(💡/🗓️)를 텍스트 보조로만 사용하고 의미 전달은 텍스트가 담당(접근성).
