🎟️쿠폰 마케팅·CRM coupon
📍 핀 4개
1개요 · 쿠폰 만들기
기능·목적
  • 화면 헤더 + 쿠폰 생성 진입 영역. 페이지 제목 쿠폰, 보조문구 "손님에게 발급할 쿠폰을 만들고 사용 현황을 봐요.", 우측 상단 [+ 쿠폰 만들기] 기본 버튼으로 구성한다.
  • [+ 쿠폰 만들기] 클릭 시 쿠폰 생성 모달을 띄워 신규 coupon 엔터티 1건을 만든다. 이 화면에서 쿠폰을 발행/관리하는 단일 진입점이다.
  • 발행처(매장/소프트먼트)에 따라 정산 처리가 달라지므로, 사장님이 직접 만드는 쿠폰은 항상 issuer='merchant'로 생성된다(소프트먼트 발행분은 본사가 별도 주입).
동작·상태값
  • 생성 모달 입력 필드: 이름(name, 텍스트), 코드(code, 영문대문자+숫자, 미입력 시 자동 생성), 종류(kind 라디오: 정액/비율/무료/증정), 할인값(value — 정액=원, 비율=%, 무료·증정=0 고정·입력 숨김), 사용 조건(condition, 예: 최소 주문금액 1만원 이상), 발급 수량(issued 정수), 사용 기간(period 시작~종료일 또는 "상시").
  • 모달 푸터 [취소]는 변경 폐기 후 닫기, [발급](기본 버튼)은 검증 통과 시 목록 맨 위에 추가하고 모달을 닫는다(spec/08-common 모달 규칙).
  • 기본값: 종류=정액, 발급 수량=무제한(빈값 허용), 기간 시작=오늘(영업일 03:00 기준 당일, SPEC §1.1), 상태=시작일이 미래면 예약·당일이면 사용중.
  • 발급 직후 토스트 "쿠폰을 발급했어요."를 띄우고 핀2 지표·핀3 목록을 즉시 갱신한다.
데이터·연동
  • coupon{ id, name, code, kind('정액'|'비율'|'무료'|'증정'), value:number, condition:string, period:string, issued:number, used:number, status('사용중'|'종료'|'예약'), issuer('merchant'|'softment') }.
  • 신규 생성 API는 POST /coupons 성격. code는 매장 내 유일(중복 불가).
  • 기간 만료(종료일 < 영업일 당일) 시 배치/조회 시점에 status='종료'로 자동 전환(자동 종료, 핀4와 동일 규칙).
  • 발행처 기본 issuer='merchant' → 정산 제외(사장님 할인 disc로 매출 차감, SPEC §2.1·정산 STATES §8). issuer='softment'는 본사 발행분으로 정산 시 보전.
예외처리·UX문구
  • 권한: 쿠폰 생성·발급은 owner 전용(SPEC §1.6). staff 계정은 [+ 쿠폰 만들기] 버튼을 비활성(disabled)하고 "사장님만 쿠폰을 만들 수 있어요." 툴팁 노출.
  • 필수값 누락 시 폼 검증: 이름 빈값 "쿠폰 이름을 적어 주세요.", 정액 value≤0 "할인 금액을 1원 이상 입력해 주세요.", 비율 value 1~100 벗어남 "할인율은 1~100% 사이로 입력해 주세요.".
  • 코드 중복 시 "이미 사용 중인 코드예요. 다른 코드를 적어 주세요.".
  • 기간 종료일이 시작일보다 빠르면 "종료일은 시작일 이후로 정해 주세요.". 발급 수량을 비우면 "수량을 비워 두면 무제한으로 발급할 수 있어요." 안내(가능형).
2쿠폰 지표
기능·목적
  • 쿠폰 KPI 4지표 카드 행. 좌→우 사용중 쿠폰·발급 누계·사용 누계·사용률을 한눈에 보여준다.
  • 쿠폰 운영 성과(얼마나 발급했고 실제 사용으로 이어졌는지)를 요약해 다음 쿠폰 기획 판단을 돕는다.
동작·상태값
  • 사용중 쿠폰 = COUPONS.filter(c=>c.status==='사용중').length, 접미사 "개".
  • 발급 누계 = sum(c.issued) 전체 쿠폰 발급 수량 합.
  • 사용 누계 = sum(c.used), 파란색(--blue) 강조.
  • 사용률 = round(sum(used)/sum(issued)*100) %. 분모 0이면 0% 표시. 4개 지표는 현재 상태 필터(핀3)와 무관하게 항상 전체 기준으로 집계(요약 성격).
데이터·연동
  • 집계 소스는 발급·사용 로그(coupon.issued, coupon.used). 카운트성 읽기 전용 지표로 API는 GET /coupons/summary 성격.
  • used는 실제 결제에 적용된 쿠폰 건수로, 결제/주문 화면의 order.couponName/couponAmt 사용 이벤트와 동기화된다.
  • 사용률 계산식은 항상 정수 반올림. 발급 무제한(issued=null) 쿠폰은 발급 누계·사용률 분모에서 제외하고 사용 누계에만 반영.
예외처리·UX문구
  • 쿠폰이 0건이면 모든 카드 0 표시(사용률 0%), "아직 만든 쿠폰이 없어요. 쿠폰을 만들어 손님에게 혜택을 줄 수 있어요." 류의 안내는 핀3 빈 상태에서 처리.
  • 분모 0 방어로 사용률 NaN/Infinity 미발생(0% 고정).
  • 권한 무관 owner·staff 모두 조회 가능(읽기 전용 지표).
3쿠폰 목록
기능·목적
  • 쿠폰 목록 패널. 제목 "쿠폰 목록" 옆 총 개수(N개)와 우측 상태 필터 칩을 제공한다.
  • 상태별로 쿠폰을 빠르게 추려보고 개별 쿠폰 행(핀4)으로 진입하는 목록 컨테이너.
동작·상태값
  • 상태 필터 칩 4종: 전체(기본 on)·사용중·종료·예약. 클릭 시 state.couponStatusFilter 설정 후 재렌더, 선택 칩에 on 강조.
  • 전체/미설정이면 모든 행, 그 외엔 c.status===필터 행만 노출.
  • 표 헤더 컬럼: 이름 | 코드 | 종류 | 조건 | 기간 | 발행처 | 사용 현황 | 상태 | (관리). 헤더 클릭 정렬 지원(SPEC §2.6 — 기본 정렬은 최신 생성/사용중 우선).
  • 각 행 클릭(또는 행 내 [관리] 버튼) 시 해당 쿠폰 상세/관리로 진입.
데이터·연동
  • 소스 COUPONS 배열, 필터 후 행 렌더. 목록 조회 API는 GET /coupons?status= 성격.
  • 헤더의 총 개수 N개는 필터 적용 전 전체 개수(COUPONS.length) 기준.
  • 상태값 status 도메인: 사용중·종료·예약. 종료·예약은 pill 색으로 시각 구분(사용중=done, 그 외=hold).
예외처리·UX문구
  • 전체 0건 빈 상태: "아직 만든 쿠폰이 없어요. 위 [+ 쿠폰 만들기]로 첫 쿠폰을 만들 수 있어요." (능동·가능형).
  • 특정 필터 결과 0건: "이 상태의 쿠폰이 아직 없어요." 안내 후 [전체 보기] 칩으로 복귀 유도.
  • 정렬/검색/페이징은 공통 규칙(SPEC §2.6) 준수, 목록 길어지면 페이지네이션 노출.
4쿠폰 행
기능·목적
  • 개별 쿠폰 행. 이름·코드·종류·조건·기간·발행처·사용 현황·상태를 한 줄에 표기하고 우측 [관리] 버튼을 둔다.
  • 발행처(매장 vs 소프트먼트)에 따른 정산 영향까지 사장님이 행에서 바로 인지하도록 한다.
동작·상태값
  • 종류 표기: 정액 → "정액 5,000원", 비율 → "비율 15%", 무료/증정 → 라벨만.
  • 사용 현황: used/issued · rate% 텍스트 + 진행 막대(width=rate%, 파란색). rate=round(used/issued*100), issued 0이면 0%.
  • 발행처 배지: issuer==='softment' → 초록 "소프트먼트" + 보조 "정산 보전"; 'merchant' → 황색 "매장 발행" + 보조 "정산 제외".
  • 상태 pill: 사용중=done(초록), 종료·예약=hold(회색). [관리] 버튼으로 수정/중단/삭제 진입(owner 전용).
데이터·연동
  • 행 필드: name, code(monospace), kind, value, condition, period, issuer, used, issued, status.
  • 소프트먼트 발행(issuer='softment') 쿠폰은 정산 시 보전: 정산 내역 화면의 couponPlatformAmt(보전금, 정산에 가산)로 연동. 매장 발행(merchant)은 couponMerchantAmt 참고용으로 정산 제외이며 사장님 할인 disc로 당일 매출 차감(정산 STATES §8).
  • 환불 발생 시 해당 거래의 쿠폰 사용은 원거래일 매출/보전에서 차감(SPEC §1.5). 결제 완료(Payment.status=paid) 거래의 취소는 결제 취소(환불)로 처리되어 쿠폰 사용 카운트·보전금도 함께 되돌린다.
  • 주문/결제 화면 쿠폰 적용 이벤트(order.couponName, couponAmt, couponIssuer)와 used 누계가 동기화된다.
예외처리·UX문구
  • 기간 만료 시 자동 종료: 종료일 < 영업일 당일이면 조회 시점에 status='종료' 전환, pill 회색. [관리]에서 재발급 안내 "기간이 끝난 쿠폰이에요. 기간을 늘려 다시 사용할 수 있어요.".
  • 예약 쿠폰(시작일 미래)은 손님 사용 불가, 상태 "예약" + "YYYY-MM-DD부터 쓸 수 있어요" 보조문구.
  • 발급 수량 소진(used>=issued, 무제한 제외) 시 사용 막대 100%·"모두 사용했어요" 표기, 신규 사용 차단.
  • [관리]는 owner 전용 — staff는 비활성 + "사장님만 쿠폰을 관리할 수 있어요." 툴팁.
