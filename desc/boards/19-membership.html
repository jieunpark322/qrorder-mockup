👑멤버십 마케팅·CRM membership
📍 핀 4개
1개요 · 등급 추가
기능·목적
  • 멤버십 화면 상단 헤더 + 등급 추가 진입점. 매장의 등급 체계(누적 결제 금액 기반 자동 등급) 전체를 관리하는 화면의 시작 영역.
  • 크럼브 마케팅 · CRM › 멤버십, 제목 멤버십, 보조문구 "누적 결제 금액에 따라 자동으로 등급이 올라가요." 노출.
  • 우상단 + 등급 추가 버튼으로 새 등급 단계를 추가(등급 체계 확장)하는 진입점 제공.
동작·상태값
  • + 등급 추가 버튼 클릭: 등급 추가 모달을 연다(목업은 alert('멤버십 등급 추가 (와이어프레임)') 플레이스홀더 — 정식 구현은 모달). 입력: 등급명(필수), 승급 기준 누적 결제액(원, 정수), 혜택 텍스트. 색상은 팔레트에서 자동/선택.
  • 저장 시 MEMBERSHIP.tiers 배열에 추가하고 threshold 오름차순으로 자동 재정렬 후 화면 재렌더. 신규 등급 members 초기값 0.
  • 등급 추가/수정/삭제는 매월 1일 자동 산정(핀 4)에 반영되며, 등급 순서는 항상 threshold 오름차순으로 유지(가장 낮은 단계 = 누적 0원 시작).
  • 권한: 기본값 사장님 전용. 권한은 추후 메뉴·카테고리별로 자유롭게 설정(직원 노출 여부를 메뉴 단위로 지정)하는 기능으로 관리 예정. 권한 없는 항목은 좌측 메뉴에 회색+자물쇠로 노출(클릭 시 403) — SPEC §1.6.2 원칙.
데이터·연동
  • 엔터티 MEMBERSHIP.tiers[] = {name:string, color:hex, threshold:int(원,≥0), benefit:string, members:int}.
  • 추가 시 검증: 등급명 필수·공백불가·중복불가(핀 3 rename과 동일 규칙), threshold는 0 이상 정수, 동일 threshold 중복 금지.
  • 저장 API 성격: POST /membership/tiers (owner 토큰 필요). 변경은 즉시 손님 앱의 등급 표시·적립율 계산에 연동.
  • members 집계는 회원 누적 결제액 배치(핀 4)가 채우는 읽기 전용 파생값 — 추가 시점에는 0.
예외처리·UX문구
  • staff가 URL /membership 직접 접근 시 접근 차단 후 안내: "이 화면은 사장님 계정에서만 볼 수 있어요." (SPEC §1.6).
  • 등급명 빈값 저장 시도: "등급 이름을 입력해 주세요." / 중복 시: "이미 사용 중인 등급 이름이에요."
  • threshold 음수·비정수 입력 시: "승급 기준 금액은 0원 이상 숫자로 입력해 주세요."
  • 저장 통신 실패 시 토스트 "등급을 저장하지 못했어요. 잠시 후 다시 시도해 주세요." (spec/08-common 토스트·에러).
2회원 · 등급 분포
기능·목적
  • 회원 · 등급 분포 KPI 카드 줄. 전체 회원 수와 등급별(SILVER/GOLD/PLATINUM 등) 회원 수를 한눈에 보여주는 요약 지표.
  • 4열 그리드: 첫 카드 전체 회원 + 등급 수만큼 등급별 카드. 사장님이 회원 규모와 등급 쏠림을 빠르게 파악하도록 함.
동작·상태값
  • 전체 회원 카드: 값 = m.tiers.reduce((s,t)=>s+t.members,0), 단위 명 표기.
  • 등급별 카드: 각 등급 members 수치 표시. 카드 라벨은 등급 색상(t.color)·굵게, 테두리는 등급 색상 40% 투명도(${t.color}40).
  • 등급 수에 따라 카드 개수 가변(등급 추가/삭제 시 카드도 증감). 등급 순서는 핀1·핀3과 동일하게 threshold 오름차순.
  • 카드는 집계 표시 전용 — 클릭 동작 없음(필터 등 인터랙션 없음). 등급 정의 편집은 핀3 표에서 수행.
데이터·연동
  • 표시값 모두 MEMBERSHIP.tiers[].members 파생. 전체 = 등급별 합. API 성격: GET /membership/summary.
  • members는 회원별 "이번 산정 주기 등급" 배치 결과의 등급별 count. 영업일/월 경계는 영업일 03:00 기준(SPEC §1.1)으로 산정한 "매월 1일" 스냅샷.
  • 등급 색상 color는 hex 문자열, 카드 테두리·라벨 색에 직접 바인딩.
예외처리·UX문구
  • 회원 0명인 신규 매장: 전체 회원 0명, 등급별 모두 0명 표시(부정문구 대신 자연스러운 0 노출).
  • 등급은 있으나 해당 등급 회원 0명: 해당 카드 0명 정상 표시(빈 상태 별도 처리 불필요).
  • 집계 로딩 중: 카드 숫자 자리에 스켈레톤/로딩 표시(spec/08-common 로딩).
  • 집계 조회 실패: 카드 영역 에러 표시 후 재시도 안내 토스트.
3등급별 혜택
기능·목적
  • 등급별 혜택 표. 각 등급의 승급 기준·혜택·회원수·비율을 한 표로 보여주고, 등급명을 편집하는 영역.
  • 패널 제목 등급별 혜택 + 부제 ${m.tiers.length}단계로 현재 등급 단계 수 표시.
동작·상태값
  • 표 컬럼: 등급 / 승급 기준 / 혜택 / 회원수 / 비율 / (편집 버튼). 등급 셀은 👑 이름 알약(pill)·등급색 배경.
  • 승급 기준 셀: 누적 {fmt(threshold)}원 이상 형식(천단위 콤마). 가장 낮은 단계는 누적 0원 이상(0원부터 시작).
  • 비율 셀: Math.round(members/totalMembers*100)%. 전체 0명이면 0%.
  • ✏️ 이름 버튼 클릭: renameMembershipTier(idx) 호출 — prompt로 새 이름 입력받아 trim 후, 빈값/동일/중복 검증 통과 시 tier.name 갱신 후 재렌더. (정식 구현은 prompt가 아닌 인라인 편집 모달로 승급기준·혜택까지 편집 권장.)
  • 행 표시 순서는 threshold 오름차순 고정.
데이터·연동
  • 행 바인딩: tier{name, threshold, benefit, members, color}. 비율 = members/Σmembers 반올림.
  • fmt()로 금액 천단위 콤마 포맷. threshold 단위는 원(정수).
  • 이름 변경 검증(코드 실측): trim 후 빈값 금지, 기존 이름과 동일하면 무변경, 다른 등급명과 중복 금지.
  • 등급 정의는 손님 앱의 적립율·혜택 노출과 연동. 저장 API 성격 PATCH /membership/tiers/{id} (owner).
예외처리·UX문구
  • 이름 빈값 입력: "등급 이름은 비워 둘 수 없어요." (코드 실측 문구).
  • 중복 이름 입력: "'{입력값}'은(는) 이미 사용 중인 등급 이름이에요." (코드 실측 문구).
  • prompt 취소(null) 시 변경 없이 종료.
  • 전체 회원 0명일 때 비율 컬럼 모두 0% 표시(0 나눗셈 가드 totalMembers?…:0).
  • 등급이 1단계만 남도록 삭제 시도 시(정식 구현): "등급은 최소 한 단계가 필요해요." 로 삭제 차단 권장.
4자동 산정 안내
기능·목적
  • 자동 산정 안내 배너. 등급이 어떤 기준·주기로 자동 산정되는지 사장님에게 설명하는 정보 영역.
  • 수동 등급 부여가 아니라 누적 결제 금액 기반 자동 산정임을 명확히 안내.
동작·상태값
  • 배너 문구(코드 실측): "💡 등급은 매월 1일 누적 결제 금액 기준으로 자동 산정돼요. 일정 기간 거래가 없으면 한 단계 떨어질 수 있어요."
  • 산정 주기: 매월 1일(영업일 03:00 기준, SPEC §1.1) 배치로 전 회원 누적 결제액을 재평가해 등급 재배치.
  • 승급: 누적 결제액이 상위 등급 threshold 이상이면 해당 등급으로 상향. 강등: 정책상 일정 기간 무거래 시 한 단계 하향.
  • 안내 전용 — 클릭/토글 없음. 정책 수치(산정일, 무거래 강등 기간)는 정식 구현에서 설정 가능 항목으로 분리 권장.
데이터·연동
  • 산정 입력값: 회원별 누적 결제 완료 금액. 정의는 Payment.status=paid 합계 기준이며, 환불은 원거래일 매출에서 차감(SPEC §1.5)하므로 누적액에서도 차감 반영.
  • 누적액 산정 시 결제 취소/환불(refunds.original_paid_at 영업일 기준 차감)을 반영 — 주문 취소(미결제)는 애초 누적에 포함되지 않음(제품노트: paid 여부로 구분).
  • 부가세 등 금액 계산은 공통식 따름(과세분 부가세 round(taxable/11), SPEC §1.3) — 누적은 결제 총액 기준.
  • 배치 결과가 tiers[].members(핀2·핀3) 및 손님 등급/적립율에 연동. API 성격: 정기 배치 recalcMembershipTiers().
예외처리·UX문구
  • 누적 0원 회원: 가장 낮은 등급(누적 0원 이상)에 자동 편입.
  • 산정 경계값(threshold와 정확히 동일 금액): "이상" 기준이므로 해당 상위 등급으로 편입.
  • 무거래 강등 기간 미설정 시: 강등 비활성(승급만 동작)으로 안전 기본값 처리 권장.
  • 환불로 누적액이 하위 기준 미만이 된 경우: 다음 산정일에 하향 반영(즉시 강등 아님) — 안내문구로 "매월 1일 기준" 명시.
