💁문의 · 자주 묻는 질문 고객 지원 support
📍 핀 3개
1개요
기능·목적
  • 화면 개요·진입점. 사장님(owner/staff)이 소프트먼트 운영팀이 정리한 사용 안내(FAQ)를 셀프로 확인하고, 답을 못 찾으면 같은 화면 안에서 서비스 관련 문의(CS)로 자연스럽게 전환하도록 하는 통합 고객지원 허브.
  • 사이드바 그룹 고객 지원, 메뉴 라벨 문의·자주 묻는 질문(icon 💁, ord 21), URL admin.qrorder.softment.co.kr/support. 진입 시 기본 탭은 faq.
  • 상단 고정 헤더: 크럼브 고객 지원 › {탭라벨}, 제목 고객 지원(h2), 그 아래 탭별 안내 부제(page-sub) 노출.
  • 셀프서비스(FAQ)를 1차로 두고 사람 응대(CS)를 2차로 두어 운영팀 문의량을 줄이는 것이 목적. 권한 무관하게 owner/staff 모두 열람·문의 가능(SPEC §1.6; 별도 권한 게이트 없음).
동작·상태값
  • 헤더 우측 액션 버튼은 탭 종속: cs 탭일 때만 + 새 문의 남기기(btn-primary) 노출, faq 탭에서는 미노출.
  • 탭 전환은 핀2 카드형 토글로만 수행. 탭 전환 시 state.supportTab 갱신 후 renderApp() 전체 리렌더.
  • 탭 상태는 state.supportTab(기본값 'faq')에 보존되어 다른 화면 갔다 돌아와도 마지막 탭 유지. navKey에 supportTab 포함되어 탭 변경도 화면 전환으로 추적.
  • 외부 진입 헬퍼: setSection('faq') → faq 탭, setSection('cs') → cs 탭으로 매핑(둘 다 section=support). openNewInquiryWithType(type,title)는 cs 탭으로 이동 후 모달을 분류·제목 미리 채워 자동 오픈.
데이터·연동
  • 엔터티 2종: FAQS[](운영팀 작성, 사장님 읽기 전용), CS_INQUIRIES[](사장님이 작성·운영팀이 답변).
  • FAQS[i] = { id:int, cat:string(시작하기·결제·정산·메뉴 관리·QR·테이블·직원 호출·환불·취소·기타), q:string, a:string }.
  • CS_INQUIRIES[i] = { id:int, date:'YYYY-MM-DD', type:string, title:string, body:string, status:'답변 대기'|'답변 완료', reply:string, replyAt:'YYYY-MM-DD HH:mm', by:string }.
  • 연동: 새 문의 등록 시 status=답변 대기 건수가 홈 대시보드 '답변 받을 문의' KPI와 동기화(inquiryWait). FAQ '직접 문의' 버튼은 cs 탭으로 라우팅.
  • API 성격(그린필드 정의): GET /faqs(운영자 콘솔에서 관리, 사장님은 read-only), GET/POST /inquiries(매장 단위 scope, store_id 바인딩). 영업일·시간 표기는 SPEC §1.1(03:00 기준) 따름.
예외처리·UX문구
  • 최초 진입·미설정 시 state.supportTab이 비어 있으면 자동으로 'faq'로 세팅(빈 상태 없이 항상 FAQ 먼저 노출).
  • FAQ 부제 인용: "소프트먼트 운영팀이 정리한 사용 안내예요. 원하는 답을 찾지 못하시면 서비스 관련 문의 탭에서 직접 남겨 주세요."
  • CS 부제 인용: "QR오더 사용 중 궁금한 점이나 문제가 있으면 남겨 주세요. 소프트먼트 운영팀이 확인 후 답변드려요."
  • 응답 SLA 안내(가능형 톤): "운영팀이 평균 24시간 안에 답변드려요." 긴급 건은 "가입 시 안내드린 CS 전화"로 유도.
2탭 (FAQ / 서비스 문의)
기능·목적
  • 탭 전환 영역. 카드형 토글 버튼 2개로 자주 묻는 질문(faq)과 서비스 관련 문의(cs)를 오가게 함. 각 버튼은 아이콘+라벨+한 줄 설명+카운트 배지로 구성된 큰 카드(.sup-tab).
  • 셀프 해결(FAQ)을 먼저, 사람 응대(CS)를 다음에 배치해 사용자가 문의 전에 FAQ를 보도록 시각적으로 유도.
  • 배지로 즉시 정보 제공: FAQ는 전체 문서 수, CS는 답변 대기 건수를 노출해 '확인해 주세요' 신호를 줌.
동작·상태값
  • 버튼1 ❓ 자주 묻는 질문 / 설명 "사용 안내를 먼저 빠르게 확인" / 배지 = FAQS.length(예: 11, 항상 표시).
  • 버튼2 📨 서비스 관련 문의 / 설명 "소프트먼트 운영팀에 직접 남기기" / 배지 = 답변 대기 건수가 0보다 클 때만 "대기 N" 표시, 0이면 배지 숨김.
  • 클릭 시 setSupportTab(key) 호출 → section=support 고정, supportTab=key 설정 후 renderApp(). 활성 탭은 .on 클래스로 강조(배지도 .on 스타일).
  • 활성 탭에 따라 본문(#support-body)이 renderFAQ() 또는 renderCS()로 교체. cs 활성 시 헤더에 + 새 문의 남기기 버튼 동시 노출.
  • 기본 활성 탭은 faq. 두 버튼은 항상 함께 노출(조건부 숨김 없음).
데이터·연동
  • faqCnt = FAQS.length — FAQ 배지 소스.
  • csWait = CS_INQUIRIES.filter(c=>c.status==='답변 대기').length — CS 배지 소스.
  • 활성 상태 플래그 state.supportTab: 'faq' | 'cs'. 외부 라우팅 키 'faq'/'cs'는 모두 section=support로 정규화(targetSection 로직).
  • CS 배지(대기 N)는 핀1의 홈 대시보드 inquiryWait KPI와 같은 계산식을 공유 → 문의 등록/답변 시 양쪽이 함께 갱신.
예외처리·UX문구
  • 답변 대기 0건이면 CS 버튼 배지를 숨겨 빈 "대기 0" 노이즈를 방지(능동·긍정 톤 유지).
  • FAQ 배지는 0이 될 일이 없도록 운영팀이 최소 1건 이상 유지하는 것을 전제(그린필드 규칙: 전 카테고리 0이면 빈 상태 문구는 핀3에서 처리).
  • 탭 라벨은 크럼브에도 반영: faq → "고객 지원 › 자주 묻는 질문", cs → "고객 지원 › 서비스 관련 문의".
  • 접근성: 토글 카드는 클릭 가능 영역 전체가 활성, 활성 탭은 색/배지로 현재 위치를 명확히 표시(spec/08-common.md 현재 위치 표기 원칙).
3FAQ 카테고리 · 목록
기능·목적
  • FAQ 카테고리·검색·아코디언 목록 영역(faq 탭 본문, renderFAQ). 운영팀이 작성한 사용 안내를 분류·검색·펼침으로 빠르게 찾게 함.
  • 카테고리: 시작하기 / 결제·정산 / 메뉴 관리 / QR·테이블 / 직원 호출 / 환불·취소 / 기타 + 맨 앞 전체. 카테고리는 FAQS의 cat에서 자동 추출(중복 제거).
  • 각 항목 클릭 시 펼쳐 답변(A.)을 보여주는 아코디언. 키워드 검색으로 질문·답변 본문을 동시 필터.
  • 하단에 '못 찾으셨나요' 안내 배너로 CS 전환을 한 번 더 유도(셀프 → 사람 응대 깔때기 완성).
동작·상태값
  • 검색: 상단 입력창(placeholder "🔍 궁금한 키워드로 검색해 보세요 (예: 정산, 옵션, QR)"), oninput마다 state.faqSearch 갱신·리렌더. 매칭은 (q+' '+a).toLowerCase().includes(검색어) 부분일치.
  • 카테고리 칩: 칩마다 건수 표시(전체=FAQS.length, 그 외=해당 cat 개수). 클릭 시 state.faqCat 변경·리렌더, 활성 칩은 .on 강조. 카테고리+검색은 AND 결합 필터.
  • 아코디언: 행 클릭 시 state.faqOpen(Set, id 보관)에 토글 추가/삭제. 펼침 시 ▾ 아이콘 180° 회전, 본문 영역에 A. 답변 노출(다중 항목 동시 펼침 허용).
  • 정렬: FAQS 원본 배열 순서 그대로(id 오름차순) 표시. 기본 카테고리 전체, 기본 검색어 빈 문자열, 기본 펼침 상태 모두 닫힘.
  • 하단 배너/빈 상태 버튼 📨 직접 문의 남기기 / 📨 서비스 관련 문의 남기기 클릭 시 setSection('cs')로 CS 탭 이동.
데이터·연동
  • 소스 FAQS[](읽기 전용). 필터 결과 rows는 cat 필터 후 search 필터 순차 적용.
  • UI 상태 플래그: state.faqCat:string(기본 '전체'), state.faqSearch:string(기본 ''), state.faqOpen:Set(펼친 항목 id).
  • 카테고리 목록 = ['전체', ...new Set(FAQS.map(f=>f.cat))] — 자동 생성이므로 운영팀이 신규 cat 추가 시 칩도 자동 노출.
  • 답변 본문은 white-space:pre-wrap으로 줄바꿈 보존. 검색·카테고리는 클라이언트 필터(그린필드 API에서는 GET /faqs?cat=&q= 서버 필터로 확장 가능, SPEC §2.6 검색·정렬 표준).
예외처리·UX문구
  • 검색/카테고리 결과 0건 빈 상태: 아이콘 🔍 + "조건에 맞는 답변이 없어요" + 안내 "다른 키워드로 검색해 보시거나, 아래 버튼을 눌러 직접 문의해 주세요." + 📨 서비스 관련 문의 남기기 버튼.
  • 하단 상시 배너: "더 궁금한 점이 있으신가요?" / "소프트먼트 운영팀이 평균 24시간 안에 답변드려요." / 버튼 "📨 직접 문의 남기기".
  • 검색어는 대소문자 무시(toLowerCase) — 영문/한글 혼용 키워드도 매칭. 공백만 입력 시 전체 노출(빈 검색과 동일 처리).
  • FAQ는 사장님 읽기 전용 — 편집·삭제 액션 없음(운영자 콘솔에서만 관리). 경계: 카테고리 전체 0건이면 빈 상태 문구가 그대로 노출되며 CS 전환 버튼이 유일한 출구.
