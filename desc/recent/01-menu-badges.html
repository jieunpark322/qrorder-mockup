메뉴 관리

01 · 메뉴 배너 (인기·추천·신메뉴)

메뉴마다 손님 메뉴판에 노출할 배너(🔥 인기 / ⭐ 추천 / 🆕 신메뉴)를 토글로 켜고 끄는 기능. 복수 선택 가능.

📍 라이브 보기 진입 경로 — 메뉴 관리 → 메뉴판 → 메뉴 카드 클릭 관련 파일 — index.html
📷 화면 캡처 · 메뉴 정보 수정 모달 — 섹션 1(기본 정보)의 메뉴 배너 토글 영역
변경 전
메뉴마다 인기·추천·신메뉴를 표시하는 기능이 없었음. 매장에서 자체적으로 메뉴명에 (인기), (추천) 같은 문구를 직접 적거나, 카테고리로 분리하는 방식.
변경 후
메뉴 정보 수정 모달의 「메뉴 배너」 영역에서 3종 배너를 토글로 ON/OFF. 복수 선택 가능. 손님 메뉴판 카드의 메뉴명 옆에 배지로 노출.

주요 요소

1
메뉴 배너 토글 (3종)
메뉴 수정 모달 · 섹션 1 (기본 정보)
🎯 기능·목적
  • 인기/추천/신메뉴 3종 배너를 메뉴마다 선택해서 손님 메뉴판에 노출
  • 매장이 강조하고 싶은 메뉴를 시각적으로 어필 (객단가·신규 메뉴 인지 향상)
  • 여러 배너 동시 선택 가능 (예: 아이스 아메리카노 — 인기 + 추천)
⚙️ 동작·상태값
  • 3개 칩 형태로 노출: 🔥 인기 · ⭐ 추천 · 🆕 신메뉴
  • 클릭 시 토글 — 켜지면 색 채워짐 + ✓ 표시, 꺼지면 회색 테두리
  • 모달 저장 시 m.badges 배열에 키 저장. 신규 메뉴 등록 시 기본 []
  • 메뉴 카드 그리드에서 메뉴명 아래 작은 배지로 노출 (배너 칠과 동일 색)
🔌 데이터·연동
  • MENUS[id].badges: string[] — 예) ['hot','reco']
  • 키 정의: MENU_BADGES = [{key:'hot',label:'인기',ico:'🔥',color,bg}, ...]
  • 저장: saveMenuEdit에서 m.badges = [...f.badges]
  • 토글 함수: toggleMenuBadge(key) → state._menuEdit.badges 갱신 → 모달 재렌더
  • 손님 메뉴판 미리보기에도 동일 배지 노출 (메뉴 카드 우측 상단)
⚠️ 예외·UX 문구
  • 모든 배너 OFF면 메뉴명 옆에 아무 표시 안 함
  • 안내 문구: "손님 메뉴판에 메뉴 이름 옆에 배너가 표시돼요. 여러 개 선택할 수 있어요."
  • 품절·숨김 상태와 무관하게 배너 설정은 유지 (다시 판매 시 자동 복원)
  • 옛 데이터(badges 필드 없음)는 ensureMenuExtras가 빈 배열로 초기화
