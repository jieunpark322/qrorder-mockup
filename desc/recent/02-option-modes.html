02 · 옵션 그룹 모드 + 최대 선택 개수

옵션 그룹의 선택 방식을 「1개만 / 여러 개」로 구분하고, 여러 개 모드일 때 그룹 자체의 최대 선택 항목 수를 지정. (예: 「샷 추가」 항목은 최대 3개까지 동시에 고를 수 있게)

📍 라이브 보기 진입 경로 — 메뉴 관리 → 옵션 그룹 → + 새 옵션 그룹 추가 (또는 ✏️ 수정)
📷 화면 캡처 · 옵션 그룹 → + 새 옵션 그룹 추가 펼침 — 선택 방식 select + 최대 선택 개수 input
변경 전
선택 방식이 1개 선택 / 중복 선택 2종이고, 중복 선택은 몇 개까지 제한할 수 없었음. 손님이 한 그룹에서 무제한으로 항목을 골라 담을 수 있어 가격·재료 통제가 어려움.
변경 후
선택 방식을 1개만 / 여러 개로 정리. 여러 개일 때만 최대 N개까지 input이 표시돼서 그룹 단위 한도를 직접 지정.

주요 요소

1
선택 방식 select (1개만 / 여러 개)
옵션 그룹 등록 카드 + 수정 모달
🎯 기능·목적
  • 옵션 그룹의 손님 선택 방식을 단순 2가지로 정의
  • 1개만 — 라디오처럼 단일 선택 (예: 사이즈 Regular/Large 중 하나)
  • 여러 개 — 체크박스처럼 복수 선택 (예: 토핑 여러 개)
⚙️ 동작·상태값
  • select 변경 시 onchange에서 c.mode 갱신
  • 여러 개 → 1개만 변경 시 maxSelect 자동으로 1로 리셋 + 입력 칸 숨김
  • 1개만 → 여러 개 변경 시 maxSelect=2로 자동 초기화 (최소값)
🔌 데이터·연동
  • OPT_GROUPS[id].mode: '1개만' | '여러 개'
  • OPT_GROUPS[id].required: boolean (선택 vs 필수)
  • 옛 값 '1개 선택' · '중복 선택'은 자동 마이그레이션 안 함 — 저장 시 새 값으로 덮어씀
⚠️ 예외·UX 문구
  • 선택 방식 select 라벨: "선택 방식" / 옵션: 1개만 선택, 여러 개 선택
  • 필수 select 라벨: "선택 필수" / 옵션: 선택 (건너뛰기 가능), 필수 (반드시 선택)
  • 옵션 그룹 목록 표에 모드 표기: 1개만 / 여러 개 (최대 3개)
2
최대 선택 개수 input (여러 개 모드 한정)
선택 방식 = 「여러 개」일 때만 노출
🎯 기능·목적
  • 그룹 단위에서 손님이 최대 몇 개 항목까지 동시에 선택할 수 있는지 제한
  • 예) 토핑 그룹에 5개 항목이 있어도 「최대 3개」로 제한 → 손님은 3개까지만 체크 가능
  • 옵션 항목별 추가 횟수(메뉴 측 한도)와 별개 — 그룹 자체의 한도
⚙️ 동작·상태값
  • input type=number, min=2, max=20
  • 표기: "최대 선택 개수 [N] 개까지"
  • 여러 개 모드일 때만 화면에 표시, 1개만 모드면 input 숨김 + 값 자동 1
  • 잘못된 값 입력 시: Math.max(2, Math.min(20, Number||2)) 자동 보정
🔌 데이터·연동
  • OPT_GROUPS[id].maxSelect: number (1개만=1, 여러 개=2~20)
  • 등록 카드 입력: state.optBulkCards[ci].maxSelect
  • 수정 모달: state._optEdit.maxSelect
  • 저장: saveOptBulk / saveOptEdit — 모드 검증 후 보정값 저장
⚠️ 예외·UX 문구
  • 모드 = 1개만이면 「최대 선택 개수」 영역 자체가 사라짐
  • 등록 목록 표 라벨: 여러 개 (최대 3개)
  • 설명: "손님이 고를 수 있는 항목 수"
  • 저장 시 maxSelect가 항목 수보다 크면 알림 없이 그대로 저장 (사장님 의도 존중)
