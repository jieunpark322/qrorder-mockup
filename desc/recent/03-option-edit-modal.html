03 · 옵션 그룹 수정 버튼 + 수정 모달

등록된 옵션 그룹을 사장님이 직접 수정할 수 있는 진입점. 기존엔 삭제 버튼만 있어서 항목 1개 바꾸려면 삭제 후 재등록이 필요했음.

📍 라이브 보기 진입 경로 — 메뉴 관리 → 옵션 그룹 → 등록된 옵션 그룹 표의 ✏️ 수정 클릭
📷 화면 캡처 · 옵션 그룹 수정 모달 — ✏️ 수정 버튼으로 진입
변경 전
등록된 옵션 그룹 표 액션 컬럼에 ✕ 삭제 버튼만 있었음. 항목 가격을 바꾸거나 새 항목을 추가하려면 그룹을 삭제하고 다시 등록 필요.
변경 후
✏️ 수정 버튼 추가. 클릭 시 전용 모달에서 그룹 정보(코드·이름·모드·필수·최대 선택 개수·항목 목록)를 그대로 편집. 저장 시 OPT_GROUPS 즉시 갱신.

주요 요소

1
✏️ 수정 버튼
옵션 그룹 페이지 · 등록된 옵션 그룹 표 · 액션 컬럼
🎯 기능·목적
  • 등록된 옵션 그룹 1건의 정보를 그대로 불러와 수정 모달 진입
  • 삭제 후 재등록 없이 가격·이름·항목 손쉽게 변경
⚙️ 동작·상태값
  • 표의 액션 컬럼에 [✏️ 수정] [✕] 가로 배치
  • 클릭 → editOpt(id) 호출 → state._optEdit에 그룹 복사 → renderOptEditModal()
  • 모달 닫기·취소·저장 후 자동 표 갱신
🔌 데이터·연동
  • 대상: OPT_GROUPS[id] (해당 그룹 1건)
  • 임시 상태: state._optEdit = {id, code, name, mode, maxSelect, required, items[], links[]}
  • 편집 영역과 분리 — 취소 시 원본 그대로 유지
⚠️ 예외·UX 문구
  • 버튼 라벨: ✏️ 수정 (파란색) / ✕ (빨간색)
  • title 속성: "수정" / "삭제"
  • 일괄 선택 모드에서도 동작 (개별 행의 수정 버튼)
2
옵션 그룹 수정 모달
전용 모달 (등록 카드 UI를 한 그룹 편집에 맞게 단순화)
🎯 기능·목적
  • 옵션 그룹의 모든 속성을 한 화면에서 편집
  • 코드·이름·필수 여부·선택 방식·최대 선택 개수·항목(이름+가격) 변경
  • 연결된 메뉴별 최대 선택 개수도 같이 편집 (옵션 그룹 단독 입력처)
⚙️ 동작·상태값
  • 모달 헤더: "✏️ 옵션 그룹 수정"
  • 입력 변경 시 state._optEdit 실시간 갱신 (저장 전까지 원본 보존)
  • 여러 개 모드일 때만 「최대 선택 개수」 input 노출
  • 「+ 옵션 항목 추가」 버튼으로 항목 행 추가
  • 저장 — saveOptEdit() → 검증 후 OPT_GROUPS[id] 직접 덮어쓰기
🔌 데이터·연동
  • 임시: state._optEdit = {...OPT_GROUPS[id]}
  • 항목 구조: items: [{name, price}]
  • 저장: 검증 통과 시 g.code/name/mode/maxSelect/required/items 일괄 갱신
  • 닫기 시 state._optEdit = null 정리
⚠️ 예외·UX 문구
  • 그룹명 빈값 → alert('옵션 그룹명을 입력해 주세요.')
  • 유효 항목 0개 → alert('옵션 항목을 1개 이상 입력해 주세요.')
  • 1개만 모드는 maxSelect 자동 1, 여러 개는 최소 2
  • 저장 성공 토스트: '${name}' 옵션 그룹을 저장했어요
