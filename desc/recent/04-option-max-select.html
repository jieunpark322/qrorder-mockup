04 · 옵션 최대 선택 개수 (옵션 그룹에서 메뉴별 설정)

옵션 그룹 수정 모달의 「🎚️ 메뉴별 최대 선택 개수」 섹션에서 — 연결된 메뉴를 선택하고 각 옵션 항목을 손님이 몇 번까지 추가할 수 있는지 설정. 옵션 그룹 쪽에서만 입력(메뉴 정보 수정 모달에는 옵션 관련 UI 없음).

📍 라이브 보기 진입 경로 — 메뉴 관리 → 옵션 그룹 → ✏️ 수정 → 🎚️ 메뉴별 최대 선택 개수
📷 화면 캡처 · 옵션 그룹 수정 모달의 「🎚️ 메뉴별 최대 선택 개수」 섹션 — 연결된 메뉴 카드별로 옵션 항목 횟수 입력
변경 전
옵션 항목은 그룹 단위로만 정의돼서, 「샷 추가는 아메리카노에서만 3회까지」처럼 메뉴마다 다른 한도를 줄 수 없었음.
변경 후
옵션 그룹 수정 모달 안에서 연결된 메뉴마다 각 항목 횟수 설정. 메뉴 정보 수정 모달에서는 옵션 관련 UI를 노출하지 않음 — 옵션 그룹 쪽이 단일 입력처.

주요 요소

1
메뉴 카드 + 옵션 항목 횟수 input
옵션 그룹 수정 모달 · 「🎚️ 메뉴별 최대 선택 개수」 섹션
🎯 기능·목적
  • 옵션 그룹이 연결된 모든 메뉴를 한 화면에서 일괄 관리
  • 옵션 1개를 여러 메뉴에 적용했을 때 메뉴마다 다른 한도 부여
  • 「샷 추가」 — 아메리카노 3회 / 디카페인 라떼 1회 / 키즈 음료 0회 같은 조합을 한 모달에서 처리
⚙️ 동작·상태값
  • 옵션 그룹의 links[]에 있는 모든 메뉴를 카드로 나열
  • 각 메뉴 카드 — 메뉴명 + 옵션 항목별 횟수 input
  • 입력 변경 시 즉시 저장 (저장 버튼 불필요)
  • 각 메뉴 카드에 「메뉴 정보 수정 →」 진입 링크 (이름·가격 등 메뉴 자체 정보 수정용)
  • 1개만 선택 모드는 안내 메시지만 노출 (항목별 1회 고정)
🔌 데이터·연동
  • 저장 위치: MENUS[menuId].optMaxQty[groupId][itemIdx]
  • 구조: { groupId: { itemIdx: maxQty } } · 키는 문자열
  • 입력 함수: setOptGroupMenuMaxQty(menuId, groupId, itemIdx, val)
  • markUnsavedUI() 호출로 상단 저장 알림 표시
  • 기본값: 데이터 없으면 1회로 표시
⚠️ 예외·UX 문구
  • 연결 메뉴 0개 — "🔗 아직 이 옵션 그룹을 적용한 메뉴가 없어요." 안내
  • 1개만 모드 — "🎚️ 메뉴별 최대 선택 개수 — 1개만 선택 모드라 항목별로 1회씩 고정돼요."
  • 입력 범위 1~10 → 벗어나면 Math.max(1, Math.min(10, ...)) 자동 보정
  • 안내: "같은 옵션 그룹이라도 메뉴마다 다르게 정할 수 있어요. 예) 샷 추가 — 아메리카노 3회, 디카페인 라떼 1회, 키즈 음료 0회"
설계 원칙 — 옵션 항목 추가 횟수는 「옵션 그룹 ↔ 메뉴」 다대다 관계 위에 얹히는 보조 데이터예요. 양쪽에서 편집 가능하면 어디서 바꾼 값이 진실인지 혼란이 생기니, 옵션 그룹 쪽 한 곳에서만 입력합니다. 같은 옵션 그룹을 여러 메뉴에 적용했을 때 메뉴마다 한도를 다르게 주는 일은 옵션 모달 한 화면에 묶어 일괄로 처리하는 편이 효율적이에요. 메뉴 정보 수정 모달에는 옵션 관련 UI를 노출하지 않습니다.
