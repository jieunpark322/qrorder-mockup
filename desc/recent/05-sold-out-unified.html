05 · 품절 처리 통합 (일시/종일 구분 제거)

옛 「일시 품절·종일 품절」 구분을 단일 「품절」로 통합. 사장님은 토글 하나만 누르면 바로 품절·해제 가능.

📍 라이브 보기 진입 경로 — 메뉴 카드의 「품절」 액션 버튼 / 오늘의 준비량 페이지
📷 화면 캡처 자리 — 메뉴 카드의 액션 영역 (수정/품절/✕) / 오늘의 준비량 페이지의 「오늘 품절 처리」 탭.
변경 전
일시 품절(잠시 멈춤) vs 종일 품절(오늘 끝까지) 2종 구분. 사장님이 어떤 걸 골라야 하는지 매번 판단 필요. 데이터 모델에 soldOutKind: 'temp'|'full' 필드 존재.
변경 후
「품절」 단일 상태로 통합. 한 번 누르면 품절·다시 누르면 판매 재개. soldOutKind 필드 폐기. 카운트·필터·표 모두 단일화.

주요 요소

1
메뉴 카드 품절 토글
메뉴판 페이지 · 각 메뉴 카드 액션 영역
🎯 기능·목적
  • 한 번 클릭으로 메뉴를 품절 상태로 전환 (즉시 손님 화면에 반영)
  • 다시 클릭하면 품절 해제 (판매 재개)
  • 「일시 vs 종일」 판단 부담 제거
⚙️ 동작·상태값
  • 버튼 라벨: 판매 중일 때 품절 / 품절 상태일 때 품절 해제
  • 품절 상태 — 빨간 배경(#fee2e2) + 빨간 글씨
  • 클릭 — quickToggleSoldOut(id) → setSoldOut(id, mode)
  • 품절 시 메뉴 카드 전체에 .sold-out 클래스 (회색·반투명)
🔌 데이터·연동
  • MENUS[id].manualSoldOut: boolean (단일 필드)
  • setSoldOut(id, mode) — mode==='on'이면 false, 그 외 true
  • effectiveStatus(m) — manualSoldOut true면 {kind:'manual-out', label:'품절'}
  • 옛 soldOutKind 필드는 무시 (마이그레이션 불필요)
⚠️ 예외·UX 문구
  • 품절 토스트: ⛔ '${m.name}'을(를) 품절로 바꿨어요
  • 해제 토스트: ✅ '${m.name}'을(를) 판매 중으로 되돌렸어요
  • 자동 품절(수량 0)과 수동 품절은 별개로 동작 (자동은 준비량 페이지에서)
  • 품절 상태는 영업일 마감 후 자동 해제되지 않음 — 사장님이 직접 해제
2
오늘의 준비량 · 품절 처리 탭
메뉴 관리 → 오늘의 준비량 → 「오늘 품절 처리」 탭
🎯 기능·목적
  • 메뉴 카드를 일일이 찾지 않고 한 화면에서 품절·해제 일괄 관리
  • 품절 메뉴 수 · 분류 필터 적용 가능
⚙️ 동작·상태값
  • 탭 진입 — state.stockTab='soldout'
  • 탭 라벨 카운트: 오늘 품절 처리 N (수동 품절된 메뉴 수)
  • 각 메뉴 행 — 「↺ 판매 재개」 또는 「품절」 단일 버튼
  • 필터: 카테고리·대/중/소분류 적용 가능
🔌 데이터·연동
  • manualOutCnt = otherMenus.filter(m=>m.manualSoldOut).length
  • 옛 카운트(tempOutCnt·fullOutCnt·displayTempOutCnt 등) 모두 제거
  • 옵션 측 동일 — setOptSoldOut(gid, idx, mode) 단일 토글
⚠️ 예외·UX 문구
  • 버튼: ↺ 판매 재개 / 품절
  • 품절 상태인 메뉴는 손님 메뉴판에서 회색 + "품절" 배지
  • 수량 관리(준비량 자동 차감)는 별개 탭 — 자동 품절은 자동, 수동 품절은 수동
마이그레이션 안내 — 옛 데이터에 m.soldOutKind 값이 있어도 코드에서는 더 이상 참조하지 않아요. 데이터를 굳이 지울 필요는 없지만, 백엔드 마이그레이션 시 함께 제거하는 게 깔끔합니다.
