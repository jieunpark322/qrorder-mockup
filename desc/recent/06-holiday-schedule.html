06 · 카테고리 판매 시간 — 법정 공휴일 처리

판매 시간 설정에서 요일·시간 외에 법정 공휴일을 별도 옵션으로 처리. (설정한 요일대로 / 공휴일엔 노출 안 함 / 공휴일에만 노출 — 3가지)

📍 라이브 보기 진입 경로 — 메뉴 관리 → 카테고리 / 메뉴 / 세트 → 판매 시간 🕒
📷 화면 캡처 자리 — 판매 시간 모달의 「🇰🇷 법정 공휴일 처리」 섹션 (요일/시간 영역 아래에 노출).
변경 전
판매 시간은 요일(월~일)과 시작·종료 시각만 지정. 공휴일을 별도로 처리할 수 없음. (예: 평일 메뉴를 공휴일에도 그대로 노출 vs 공휴일엔 다른 메뉴만 노출 — 구분 불가)
변경 후
판매 시간 모달에 「🇰🇷 법정 공휴일 처리」 섹션 추가. 한국천문연구원 공휴일 데이터를 기준으로 3가지 옵션 중 선택.

주요 요소

1
법정 공휴일 3가지 옵션 칩
판매 시간 모달 · 요일/시간 영역 아래
🎯 기능·목적
  • 요일 설정과 별개로 공휴일에 어떻게 동작할지 지정
  • 매장이 공휴일에 평일 메뉴 그대로 갈지, 다른 메뉴로 갈지 자유롭게 설정
  • 국가 지정 공휴일 + 임시 공휴일 모두 자동 인식
⚙️ 동작·상태값
  • 3가지 칩 — 설정한 요일대로 노출 / 공휴일엔 노출 안 함 / 공휴일에만 노출
  • 클릭 시 state._schDraft.holiday 갱신 + 모달 재렌더
  • 저장 시 schedule.holiday 필드로 영구 저장
  • 기본값: 'include' (설정한 요일대로 — 공휴일 영향 없음)
🔌 데이터·연동
  • schedule.holiday: 'include' | 'exclude' | 'only'
  • 적용 대상: 메뉴 (m.schedule) · 카테고리 (c.schedule) · 세트 (s.schedule)
  • openScheduleModal — 기존 schedule에 holiday 필드 없으면 'include' 자동 부여
  • 한국천문연구원 공휴일 데이터 API 연동 (구현 시점 결정)
⚠️ 예외·UX 문구
  • 안내: "한국천문연구원 공휴일 데이터를 기준으로 자동 적용돼요. (설날·추석·어린이날·삼일절 등)"
  • 임시 공휴일(대통령선거·국가추념일 등)도 데이터 들어오는 즉시 자동 반영
  • 대체공휴일도 동일 처리 (캘린더 데이터에 포함)
  • 공휴일 처리 옵션은 「판매 시간 제한」이 켜져 있을 때만 의미 있음 — 꺼져 있으면 항상 노출
옵션 의미 정리
• 설정한 요일대로 노출 (기본) — 공휴일이라도 위에서 정한 요일·시간 기준 그대로. 예: 평일 메뉴를 광복절(8/15 금요일)에도 노출.
• 공휴일엔 노출 안 함 — 공휴일에는 요일 설정과 무관하게 메뉴 숨김. 예: 직장인 점심 메뉴를 공휴일엔 안 팔기.
• 공휴일에만 노출 — 공휴일에만 메뉴 표시. 예: 공휴일 이벤트 메뉴.
