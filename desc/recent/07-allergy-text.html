07 · 알레르기 정보 자유 텍스트 입력

기존 14종 토글(우유·달걀·밀·…)을 폐기하고, 사장님이 직접 입력하는 textarea로 변경. 원산지 페이지에서 분리해 메뉴 상세 안으로 이동.

📍 라이브 보기 진입 경로 — 메뉴 관리 → 메뉴판 → 메뉴 클릭 → ⚠️ 알레르기 정보 (섹션 5)
📷 화면 캡처 · 메뉴 정보 수정 모달 — 섹션 5(알레르기 정보) 자유 텍스트 입력
변경 전
메뉴 관리 → 원산지·알레르기 페이지의 「⚠️ 알레르기 정보」 탭에 14종 항목별 체크 표. 사장님이 매번 14개 항목을 클릭. 손님 화면에는 아이콘으로 표시.
변경 후
메뉴 수정 모달 안 「⚠️ 알레르기 정보」 섹션의 textarea 한 칸. 사장님이 자유롭게 입력 (예: "우유, 대두, 견과류"). 손님 화면에 입력 텍스트 그대로 노출. 원산지 페이지는 「원산지 표시」 단일 페이지로 단순화.

주요 요소

1
알레르기 정보 textarea
메뉴 수정 모달 · 섹션 5 (알레르기 정보)
🎯 기능·목적
  • 14종 토글 학습 부담 제거 — 사장님이 평소 부르는 말로 자유 입력
  • "견과류 제조 시설", "글루텐 함유 가능" 같은 14종 외 문구도 자유롭게
  • 법적 의무 표시 책임은 사장님 (정확한 입력 안내)
⚙️ 동작·상태값
  • textarea (min-height 60px, resize:vertical)
  • placeholder: 예) 우유, 대두, 견과류 / 밀가루·달걀·우유 함유, 견과류 제조 시설
  • 저장 시 m.allergyText로 trim 후 저장
  • 줄바꿈 지원 (손님 화면도 줄바꿈 유지)
🔌 데이터·연동
  • MENUS[id].allergyText: string (default '')
  • 옛 m.allergens: string[] (코드 배열) — ensureMenuExtras(m)가 자동 변환 (예: ['milk','soy'] → '우유, 대두')
  • 변환 시 ALLERGENS 매핑 사용 ({code, label, icon})
  • 저장: document.getElementById('me-allergy').value.trim()
⚠️ 예외·UX 문구
  • 빈 값 가능 (선택 입력)
  • 안내: "알레르기 정보가 정확하지 않으면 손님에게 피해가 갈 수 있어요. 실제 식재료를 확인 후 정확하게 입력해 주세요. 손님 메뉴판에 입력하신 그대로 표시돼요."
  • HTML 입력 차단 — .replace(/</g,'&lt;')로 이스케이프 표시
2
손님 메뉴판 미리보기 표시
손님 화면 미리보기 · 알레르기 안내 영역
🎯 기능·목적
  • 메뉴별 알레르기 정보를 메뉴판 하단에 통합 노출
  • 아이콘이 아닌 텍스트라 가독성·정확성 향상
⚙️ 동작·상태값
  • 표시 영역: 메뉴판 하단 「⚠️ 알레르기 안내」 카드
  • 1줄 형식 — 아메리카노 · 우유, 대두
  • 알레르기 텍스트가 비어있는 메뉴는 안 보임
  • 공통 안내문(STORE_ALLERGY.notice)도 함께 노출
🔌 데이터·연동
  • 필터: MENUS.filter(m => m.status!=='숨김' && m.allergyText)
  • 각 메뉴 — ${m.name} · ${m.allergyText} 출력
  • STORE_ALLERGY.notice — 매장 공통 안내문 (예: "알레르기 문의는 직원에게...")
⚠️ 예외·UX 문구
  • 알레르기 입력된 메뉴 0개 + 공통 안내문도 빈값이면 안내 영역 자체가 안 보임
  • HTML 이스케이프 적용 — <는 표시
  • 긴 텍스트는 1줄로 줄바꿈 처리
3
원산지 페이지 단순화
메뉴 관리 → 원산지 표시 (옛 「원산지·알레르기」)
🎯 기능·목적
  • 원산지와 알레르기를 한 페이지에 묶지 않고 분리
  • 원산지는 매장 전체 공통 textarea만, 알레르기는 메뉴 상세로
  • 사이드바 진입 라벨도 단순화 — "원산지 표시"
⚙️ 동작·상태값
  • 옛 「알레르기 정보」 탭 제거 (탭 자체와 콘텐츠 삭제)
  • 제목 「원산지·알레르기 정보」 → 「원산지 표시」
  • 알레르기 안내 카드 추가 — "메뉴 상세에서 관리" 안내 + 「📖 메뉴판으로 가기」 버튼
🔌 데이터·연동
  • 원산지: STORE_ORIGIN.text (변경 없음)
  • 매장 알레르기 안내문: STORE_ALLERGY.notice (메뉴판 하단에서 사용)
  • 메뉴 알레르기는 MENUS[].allergyText로 이동
  • 옛 toggleMenuAllergen 함수 삭제
⚠️ 예외·UX 문구
  • 옛 14종 토글 데이터(m.allergens)는 자동 마이그레이션 (코드→한글 라벨)
  • 안내 카드: "알레르기 정보는 각 메뉴 상세 안에서 관리해요. 메뉴 관리 → 메뉴판에서 메뉴를 클릭한 뒤 알레르기 정보 항목에서…"
  • 메뉴 탭 라벨도 「🌿 원산지 표시」로 단순화
