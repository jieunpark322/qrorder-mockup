09 · 엑셀 다운로드 버튼 통일

화면마다 다른 라벨(📤 내보내기 · 📤 엑셀로 받기 · 📥 엑셀·CSV 받기)을 「📥 엑셀 다운로드」 하나로 통일. CSV 옵션 제거.

📍 라이브 보기 적용 화면 — 매출 한눈에 보기 / 매출 상세 / 주문 내역 / 정산 내역 / 결제 내역 / 메뉴판 등 전 화면
📷 화면 캡처 · 엑셀 다운로드 모달 — 매출 한눈에 보기에서 진입한 모습
변경 전
버튼 라벨이 화면마다 제각각 — 「📤 내보내기」 / 「📤 엑셀로 받기」 / 「📥 엑셀·CSV 받기」. 클릭하면 엑셀 vs CSV 형식 선택 카드 2개 노출. 사장님이 매번 선택 부담.
변경 후
전 화면 「📥 엑셀 다운로드」 단일 라벨. CSV 옵션 제거. 모달에서 조회 기간·건수만 확인 후 한 번에 다운로드.

주요 요소

1
통일된 📥 엑셀 다운로드 버튼
매출·정산·주문·메뉴 화면 우측 상단 툴바
🎯 기능·목적
  • 화면마다 다른 라벨을 통일해 사장님 학습 부담 제거
  • 「내보내기」「받기」 같은 추상적 단어 대신 직접적 「엑셀 다운로드」 사용
  • 📥 아이콘으로 "받기 = 내려받기" 시각화
⚙️ 동작·상태값
  • 버튼 — btn-primary 스타일 (파란 배경)
  • 클릭 시 openExport(rowsCount, source, dateRange) 호출
  • 각 화면 자체 다운로드 — openMenuExport() 메뉴 다운로드는 별도
  • 옛 라벨 모두 일괄 변경 — replace_all로 「📤 내보내기」 → 「📥 엑셀 다운로드」, 「📤 엑셀로 받기」 → 「📥 엑셀 다운로드」
🔌 데이터·연동
  • 호출 — openExport(rowsCount, source, dateRange)
  • 모달 상태 — state.exportRowsCount/exportSource/exportDateRange
  • 형식 — state.exportFormat = 'xlsx' 고정
  • 적용 화면: 매출 한눈에 보기 · 매출 상세 · 주문 내역 · 정산 내역 · 결제 내역 · 메뉴판 등
⚠️ 예외·UX 문구
  • title: 없음 (라벨이 충분히 명확)
  • 호버 시 색만 조금 진해짐 (#1e6cd9)
  • 안내문도 통일 — 엑셀 다운로드는 조회된 N건 전체
2
단순화된 다운로드 모달
버튼 클릭 후 열리는 모달
🎯 기능·목적
  • 형식 선택 단계 제거 — 1 step으로 단순화
  • 다운로드 직전에 조회 기간·건수 확인 (실수 방지)
  • 모달 그대로 누르면 엑셀 다운로드
⚙️ 동작·상태값
  • 헤더: 📥 엑셀 다운로드 — sub: ${source} · 조회 조건 기준 전체 N건
  • 본문 1 — 파란 카드 「📅 지금 조회한 기간」 (dateRange 있으면)
  • 본문 2 — 회색 카드 「📊 적용한 필터 기준으로 전체 N건을 엑셀(.xlsx) 파일로…」 (dateRange 없으면)
  • 버튼: [취소] [📥 엑셀 다운로드]
  • 확인 — confirmExport(cnt, src) → 토스트 + 모달 닫기
🔌 데이터·연동
  • 형식 선택 카드(exportFormatCards) 호출 제거
  • state.exportFormat = 'xlsx' 강제 설정
  • CSV 변환 함수(toCSV)는 유지하되 사용 안 함 (코드 폐기 X)
  • XLSX 라이브러리(SheetJS) 로딩 필수
⚠️ 예외·UX 문구
  • 0건 — 모달은 열리지만 안내 카드에 "전체 0건" 표시 (다운로드 가능)
  • XLSX 라이브러리 미로드 — alert('엑셀 라이브러리를 불러오는 중이에요...')
  • 완료 토스트 — 📥 [source] N건을 엑셀 파일로 내려받았어요
  • 모달 외 클릭 시 자동 닫힘
3
메뉴 내려받기 — 같은 패턴 적용
메뉴 관리 → 메뉴판 → 📥 엑셀 다운로드
🎯 기능·목적
  • 등록된 메뉴 전체를 엑셀로 내려받기 (가져오기 양식과 동일 컬럼)
  • 수정 후 그대로 재업로드 가능 (왕복 워크플로)
⚙️ 동작·상태값
  • 버튼 라벨 — 「📥 엑셀·CSV 받기」 → 「📥 엑셀 다운로드」
  • 모달 — 옛 형식 선택 카드 제거
  • 다운로드 함수: confirmMenuExport() → SheetJS로 .xlsx 생성
  • 파일명: qrorder-menu-YYYY-MM-DD.xlsx
🔌 데이터·연동
  • 대상: MENUS 전체
  • 컬럼: MENU_EXCEL_COLUMNS (가져오기 양식과 동일)
  • 워크시트: XLSX.utils.aoa_to_sheet + 컬럼 너비 조정
  • 저장: XLSX.writeFile(wb, filename)
⚠️ 예외·UX 문구
  • 안내: "받은 파일은 📊 엑셀로 한 번에 올리기 양식과 컬럼이 같아, 수정해서 그대로 다시 올릴 수 있어요."
  • title: "등록된 메뉴를 엑셀(.xlsx) 파일로 내려받을 수 있어요"
  • 완료 토스트: 📥 메뉴 N개를 엑셀 파일로 내려받았어요
일괄 변경 흔적
• 📤 내보내기 → 📥 엑셀 다운로드 (replace_all)
• 📤 엑셀로 받기 → 📥 엑셀 다운로드 (replace_all)
• 📥 엑셀·CSV 받기 → 📥 엑셀 다운로드 (메뉴판)
• exportFormatCards 호출 — renderExportModal·renderMenuExportModal에서 모두 제거
• toCSV · triggerDownload 함수는 유지 (다른 곳에서 미사용이지만 향후 필요 시 사용)
