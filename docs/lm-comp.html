UI 컴포넌트 & 모듈 명세서

화면 전체를 따로 그리지 않고 조건에 따라 변하는 컴포넌트 영역만 정의합니다. 디자이너-프론트엔드 싱크용 — 한 화면 안에서 이 블록들이 상태값에 따라 교체됩니다.

🛒
A. 장바구니 / 주문 내역 영역
메뉴·결제·완료 화면 공통
조건 A · 혼자
내가 담은 메뉴만 노출 · 합계 = 내 소계 · '주문하기' 버튼 활성화
조건 B · 함께
참여자별 담은 메뉴 실시간 노출 · 상단 참여자 바(N명) · 개별 결제=내 메뉴만, 함께 결제=전체 합산 / 누구나 결제 가능
🔔
B. 알림 / 대기 수단 영역
결제 직전 바텀 시트
조건 A · 비회원+카톡필요
카톡 로그인 / 휴대폰 직접 입력 버튼 · ✉️ 이메일로 받기(휴대폰 없는 외국인 대응) · 마케팅 동의 시 쿠폰 노출
조건 B · 전광판 호명
"대기번호 105번 · 전광판을 확인해주세요" 문구 · 연락처 입력 없음 → 시트 자체 생략
조건 C · 로그인 유지
시트 생략 · 카카오 연락처로 알림 자동 발송
조건 D · 서빙
"알림 없이 주문하기" 허용(선택) · 영수증 카톡 발송만 옵션 제공
💳
C. 결제 영역
결제 화면 하단
조건 A · 선불
결제수단 선택(카드/페이) · 쿠폰 적용 · 약관 동의 · "○○원 결제하기" → PG 진행
조건 B · 후불
결제수단·쿠폰·약관 전체 숨김 · "주문 넣기" → 전송만, 결제는 현장
조건 C · 예약
수령 시간 선택(시·분 드롭다운) 필수 · 미선택 시 버튼 비활성
✅
D. 완료 안내 영역
주문 완료 화면 · 1개 화면 / 안내 블록만 교체
조건 A · 대기번호
"대기번호 105번" 카드 + 진행 단계 바 + 알림 수단 라벨(카톡/이메일)
조건 B · 서빙
번호 카드 없음 · "자리로 가져다 드릴게요"
조건 C · 후불
🧾 "나갈 때 카운터에서 결제해 주세요" 앰버 안내 추가
조건 D · 예약
📅 "수령 예정 오늘 14:30 · 사장님 수락 후 확정" 안내
