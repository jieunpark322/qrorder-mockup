분기 매트릭스

비즈니스 로직상 존재할 수 없는 조합을 먼저 쳐내는 표입니다. 개발자가 조건문(if-else)·DB 제약을 설계할 때, 디자이너가 UI에서 선택을 차단할 지점을 잡을 때의 기준점입니다.

🚫 차단 규칙 — 만들어지면 안 되는 조합
조합판정사유 · 처리 방식
포장·예약 + 테이블 서빙❌ 불가포장·예약은 카운터 수령/픽업만 존재. 서빙 개념 없음 → 유형 선택 시 서빙 축 자체를 숨김
셀프 수령 + 후불❌ 불가미수령 리스크로 셀프는 선불 강제. setPayTiming('postpay') 차단 + toast 안내
포장·예약 + 후불❌ 불가현장 결제 대상이 아님. 항상 선불 → 후불 버튼 미노출
포장·예약·매장 + 함께 주문❌ 불가함께 주문은 같은 테이블 전제 → 매장 식사에서만 활성화
테이블 서빙 + 호출 방식(카톡/전광판)➖ 무의미직원이 자리로 가져다줌 → 대기번호·호명 불필요. 호출 축 N/A, callNo=null
포장·예약 + 전광판 호명 단독➖ 흡수대기번호(전광판)는 항상 발급되고 알림(카톡/폰/이메일)은 별도 필수 → "호명 vs 카톡" 선택지 자체가 없음

※ 호출 방식을 손님이 직접 고르는 건 매장 식사 + 셀프 수령일 때뿐입니다. 나머지는 유형이 자동 결정합니다.

✅ 유효 조합 매트릭스 — 6개 핵심 구성

로그인 상태(2)·함께 주문(매장식사 ×2)은 아래 6개 위에 얹히는 변형이라 별도 행으로 펼치지 않았습니다. 6 × 변형 = 실질 20개 조합.

예외 상황 가이드

주문·결제·네트워크·세션이 얽히는 지점의 처리 정책입니다. 필수 개발 전 확정 필요 · 권장 기본 정책 제안. 헤더를 누르면 펼쳐집니다.

👥함께 주문 동기화▼
동시 담기·수정
권장실시간 소켓(WebSocket) 브로드캐스트로 공동 장바구니 갱신. 충돌 시 last-write-wins + 항목별 누가 담았는지 표시
참여자 이탈
필수한 명이 페이지를 닫아도 다른 참여자 장바구니는 유지. 이탈자 항목은 "○○님 담음"으로 잔존, 결제 시 포함 여부 확인
방장 이탈 / 결제권
필수방장 개념 없이 누구나 결제 가능(현재 정책). 한 명이 함께 결제하면 전체 잠금 → 중복 결제 방지
🔑로그인 / 테이블 세션▼
재진입 (테이블 미초기화)
필수이전 카카오 로그인 이력 + 테이블 점유 중이면 재인증 생략, 연락처·내역 복원 (PRD §12 참조)
테이블 비우기 누락
권장POS '테이블 비우기' 깜빡 대비 → 시간 기준 만료(무활동 5분) + 손님측 "새 주문 시작" 수동 초기화 버튼 병행
세션 만료
무활동 5분 → 'expired' 화면. QR 재스캔으로만 새 세션. 다른 기기 동일 URL은 독립 세션(공유 안 됨)
⚠️결제 실패 / 중도 이탈▼
PG 창 이탈·취소
필수장바구니 유지 → 결제 화면 복귀(backToCheckout()). 주문은 PG 성공 콜백 전까지 미전송 → 테이블 점유는 유지
결제 실패 (PAYMENT_FAILED)
에러 화면 → "다른 방법으로 결제" → 결제 화면 복귀. 카드 한도/잔액 안내
POS 끊김 (POS_OFFLINE)
결제는 됐으나 주문 전송 실패 가능 → "다시 시도"(placeOrder()). 서빙 매장은 직원 호출 버튼 추가
네트워크 오류
"다시 시도" 재전송. 멱등키(idempotency key)로 중복 주문 방지 권장
점유 해제 타이밍
권장선불 미완료 이탈 시 즉시 해제하지 않고 무활동 타이머에 위임 → 손님 복귀 시 이어서 결제 가능
