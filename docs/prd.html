1. 서비스 개요

손님이 테이블 QR 또는 포장용 QR을 스캔하면 진입하는 모바일 웹 주문 화면이다. 별도 앱 설치 없이 브라우저에서 메뉴 탐색 → 옵션 선택 → 결제/주문까지 완결한다.

서비스 범위
  • 대상 사용자: 매장 방문 손님 (비회원 · 카카오 로그인)
  • 플랫폼: 모바일 웹 (max-width 480px)
  • 진입: QR 스캔 (테이블 고정 QR · 포장 QR · 포장예약 QR)
  • 세션 유지 시간: 5분 무활동 시 세션 만료 → QR 재스캔 안내

4. 화면 목록

① 메뉴 화면 menu

브랜드 헤더, 카테고리 탭, 메뉴 카드, 장바구니 바, 직원 호출 FAB

② 결제 화면 checkout

주문 내역 확인, 요청사항, 결제 수단, 쿠폰, 약관, 알림 수단 시트(오버레이)

③ 처리 중 processing

결제 중 / 주문 접수 중 스피너 표시

④ 에러 error

결제 실패 / POS 연결 끊김 / 네트워크 오류별 안내 및 재시도

⑤ 주문 완료 done

주문번호, 호출번호, 진행 상태바, 새 주문 버튼

⑥ 세션 만료 session

5분 무활동 시 노출. QR 재스캔 안내

5. 메뉴 화면 기능 명세

5-1. 브랜드 헤더

  • 대표 이미지 캐러셀 (최대 3장, 3.5초 자동 전환)
  • 주문 유형 뱃지 (먹고가기/포장/예약 + 테이블 번호)
  • 매장명, 영업 상태 표시
  • 주문 내역 버튼 (이전 주문 차수가 있을 때만 노출)
  • 쿠폰 버튼 (전화번호 10자리 + 마케팅 동의 조건 충족 시 노출)

5-2. 탭 구성

탭내용
메뉴카테고리 탭 + 메뉴 카드 목록
리뷰손님 리뷰 목록 + 평균 평점 (사장님 설정으로 on/off)
매장정보영업시간, 원산지 정보, 공지사항

5-3. 메뉴 카드

  • 카테고리별 섹션 분리, 카테고리 탭 클릭 시 해당 섹션으로 스크롤
  • 스크롤 위치에 따라 카테고리 탭 자동 활성화 (스크롤 스파이)
  • 메뉴 검색 (이름 · 설명 텍스트 검색)
  • 배지: 🔥 인기 / ⭐ 추천 / 🆕 신메뉴
  • 상태: 판매중 / 품절 (dimming + 오버레이) / 숨김 (노출 안 됨)
  • 재고 부족 (3개 이하): "잔여 N개 · 서둘러 주세요" 표시
  • 해피아워 할인 (사장님 설정 시간대 + 시뮬레이터 토글)
  • 공휴일 표시 정책: 포함(기본) / 공휴일 숨김 / 공휴일만 노출
  • 포장가: dinein이 아닐 때 takeout 가격 적용

5-4. 메뉴 상세 시트 (바텀 시트)

  • 대표 이미지, 이름, 설명, 가격 (할인 표시)
  • 알레르기 정보
  • 옵션 선택: 단일 선택(라디오) / 복수 선택(체크박스)
  • 수량 스테퍼 (최소 1, 최대 maxQty)
  • 담기 버튼 (같은 옵션 조합이면 수량 누적)

5-5. 세트 메뉴

  • 슬롯별 구성 선택 (각 슬롯은 필수 선택)
  • 기본가 + 옵션별 추가 금액 합산

5-6. 공동 주문 (먹고가기 전용)

  • 같은 테이블에 이미 주문 중인 손님이 있으면 합류 시트 노출
  • 합류 시: 참여자 아바타 바 표시, 서로의 장바구니 확인 가능
  • 장바구니 시트에서 개별 결제 / 함께 결제 선택

5-7. 직원 호출 (서빙 매장 · 영업 중 · 메뉴 탭에서만)

  • FAB 버튼으로 호출 시트 열기
  • 사유 항목 복수 선택 후 호출 (최대 15항목, 3열 그리드)
  • 기본 항목: 물, 물티슈, 수저/포크, 앞접시, 휴지, 빨대, 얼음, 추천 메뉴, 계산 도움, 아기 의자, 담요, 기타 문의

5-8. 장바구니 바 / 장바구니 시트

  • 메뉴 탭 + 영업 중 + 담은 항목 있을 때 하단 고정 바 노출
  • 장바구니 시트: 항목별 수량 조절·삭제, 공동 주문 시 참여자별 항목 표시
  • 장바구니가 완전히 비워지면 자동으로 메뉴 화면으로 복귀

6. 결제 화면 기능 명세

6-1. 주문 내용 확인

  • 담은 메뉴 목록 (수량 조절·삭제 가능, 변경 시 스크롤 위치 유지)
  • 공동 주문 + 함께 결제 시: 게스트 항목 포함하여 전체 합계 표시

6-2. 수령 시간 선택 (포장 예약 전용)

  • 12:00 ~ 20:30 범위 30분 단위 타임슬롯 선택 (필수)
  • 미선택 시 결제 버튼 비활성 → 토스트 안내

6-3. 결제 수단

  • 카드, 카카오페이, 네이버페이, 토스페이
  • 후불(postpay) 시 결제 수단 섹션 미노출

6-4. 쿠폰 섹션

조건노출 내용
전화번호 10자리 + 마케팅 동의 보유 쿠폰 목록 (라디오 선택) + 쿠폰번호 코드 입력
그 외 쿠폰번호 코드 입력만 노출
후불(postpay) 쿠폰 섹션 전체 미노출
  • 쿠폰 타입: amount(정액 할인) / rate(정률 할인, maxDisc 상한)
  • 최소 주문 금액(minOrder) 미달 시 쿠폰 비활성
  • 채널 필터: 쿠폰별 channels 배열로 주문 유형별 적용 여부 설정

6-5. 요청사항

  • 사장님이 on/off 가능 (기본 on)
  • 자유 입력 텍스트, 최대 200자

6-6. 약관 동의

약관필수/선택노출 조건
이용약관 동의필수선불 결제 시
개인정보 수집·이용 동의필수선불 결제 시
만 14세 이상 확인필수선불 + 비회원
쿠폰·이벤트 알림 수신 동의선택알림 수단 시트 인라인

6-7. 알림 수단 선택 시트 (바텀 오버레이)

결제 버튼 클릭 → 조건에 따라 시트 표시 여부 결정:

조건동작
카카오 로그인 상태시트 생략 → 바로 주문 처리
셀프수령 + 호명 방식시트 생략 → 알림 없이 주문 처리
서빙 + 후불시트 생략 → 알림 없이 주문 처리
그 외알림 수단 시트 표시

시트 내 선택지:

  • 🟡 카카오톡 연동: 마케팅 동의 체크 시 보유 쿠폰 즉시 표시
  • 📱 휴대폰번호 직접 입력: 10자리 입력 + 마케팅 동의 시 보유 쿠폰 표시
  • 알림 없이 주문하기 (알림 불필요 케이스에서만 노출)
알림 수단 시트 진입 시: S.phone과 S.termsChecked.marketing 초기화 → 매번 새로 입력.

7. 주문 처리 · 완료 · 에러 화면

7-1. 처리 중 화면

  • 선불: "결제 처리 중" → (1.5초) → "주문 접수 중" → (1.3초) → 완료/에러
  • 후불: "주문 접수 중" → (1.3초) → 완료/에러

7-2. 주문 완료 화면

  • 주문번호 (형식: #YYYYMMDD-XXXX)
  • 호출번호: 서빙 매장 제외 전 케이스 발급 (형식: 사장님 설정, 예 A-001)
  • 4단계 진행 상태바 (주문 유형별 라벨 상이)
  • 주문 내역 요약 (쿠폰 할인 표시 포함)
  • 후불 시: "나갈 때 카운터에서 결제해 주세요" 안내 pill
  • 포장예약 시: 수령 예정 시간 + "사장님 수락 후 확정" 안내
  • 새 주문하기 버튼 → 장바구니/쿠폰 초기화, 메뉴 화면으로

7-3. 상태바 단계 (주문 유형별)

유형1단계2단계3단계4단계
포장하기주문 접수준비 중픽업 대기픽업 완료
포장 예약주문 접수수락 대기준비 중완료
먹고가기 (셀프)주문 접수준비 중수령 대기완료
먹고가기 (서빙)주문 접수준비 중서빙 중완료

7-4. 에러 화면 (3종)

에러 코드제목주요 동작
PAYMENT_FAILED결제에 실패했어요결제 화면으로 돌아가 다른 수단 선택
POS_OFFLINE매장 시스템 연결 끊김재시도 / 서빙 매장이면 직원 호출 버튼 추가 노출
NETWORK_ERROR연결이 불안정해요재시도

8. 주문 내역

  • 테이블 세션 내 차수별 주문 누적 기록
  • 차수별 주문 시간, 항목, 합계 표시
  • 헤더 "주문내역 N" 버튼으로 시트 열기 (이전 차수 있을 때만 노출)

9. 세션 관리

  • 무활동 5분 후 자동으로 세션 만료 화면 전환
  • touchstart, click, scroll, keydown, input 이벤트로 타이머 리셋
  • tableActive 상태로 POS 테이블 점유 여부 구분 (false = 테이블 클리어 → 비회원 auth 초기화)

11. 비기능 요구사항

  • 스크롤 위치 보존: 화면 유지 시 render() 호출로 스크롤 위치 초기화되지 않음
  • 스크롤 잠금: 시트/모달 열림 시 body 스크롤 잠금, 닫힘 시 해제
  • 접근성: aria-label, 색 대비, 터치 타깃 크기 고려
  • 이미지 lazy loading, onerror 폴백 처리
  • 결제 버튼 금액 실시간 반영 (쿠폰 선택·해제 즉시)
