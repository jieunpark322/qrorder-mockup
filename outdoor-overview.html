야외형 QR오더 — 구조 이해 가이드

야외 행사장에서 손님이 QR을 스캔해 여러 매장의 메뉴를 주문·픽업하는 서비스입니다. 어디에 쓰이고, 어떤 구조로 돌아가는지 한눈에 정리했어요.

1한 줄 요약

여러 매장(부스·푸드트럭)이 모인 야외 행사장에서, 손님은 앱 설치 없이 QR 스캔 → 메뉴 주문 → 현장 픽업까지 한 번에. 운영 주체(브랜드)는 행사 단위로 매장·메뉴·정책을 통합 관리합니다.

4두 가지 화면 구조

행사 규모·구성에 따라 두 방식 중 선택돼요. (프로토타입 상단 토글로 비교 가능)

매장 그룹 구조 여러 매장

  • 스플래시 → 그룹 홈(배너·공지·검색·카테고리·매장 목록) → 매장 진입
  • 여러 매장을 한 화면에서 탐색
  • 담기 방식: 통합 결제(여러 매장 한 번에) / 개별 결제(매장별)

단일 매장 구조 한 매장

  • 매장 페이지가 곧 시작점 (그룹 홈 없음)
  • 단독 운영 매장·소규모 행사에 적합
  • 해당 매장 상품만 담기·결제

5설정 우선순위 — 브랜드가 매장보다 우선

대기 정보 표시를 예로, 같은 브랜드(행사)에 속하면 브랜드 설정이 매장 개별 설정을 덮어써요.

🔑 규칙 — 매장은 평소 대기 표시를 자유롭게 설정하지만, 브랜드가 이 행사에 대해 표시 방식을 설정 완료한 순간부터는 브랜드 설정이 우선 적용됩니다. 결과적으로 행사 내 전 매장이 동일한 방식으로 통일 노출돼요.

정리
· 매장 단위(평소): 대기 현황을 매장이 자유 설정 — 표시 여부 및 방식(주문 대기수 / 대기 인원 / 대기 시간).
· 브랜드 단위(행사 소속 시, 우선): 브랜드 설정 페이지에서 이 행사의 대기 정보 ① 표시 여부, ② 표시 방식(대기 인원 ↔ 대기 시간)을 지정 → 행사 내 전 매장에 우선 적용.
· 브랜드가 미표시로 두면 개별 매장이 켜 두었더라도 노출되지 않아요.
