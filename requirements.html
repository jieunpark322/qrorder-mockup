01

목적 및 범위

이 시스템이 해결하는 문제
📱

앱 설치 없는 주문 경험

손님은 QR 스캔 한 번으로 설치·로그인 없이 즉시 주문할 수 있어야 합니다. 카카오 로그인은 선택사항이며, 알림을 더 편하게 받기 위한 제안에 그쳐야 합니다.

🔀

6가지 매장 운영 방식 지원

매장 형태(매장·포장·예약) × 서빙 방식(서빙·셀프호명·셀프알림톡) × 결제 타이밍(선불·후불) 조합에서 발생하는 6개 시나리오를 단일 코드베이스로 처리합니다.

🛡️

신뢰할 수 있는 결제 흐름

중복 결제 방지, 위조 QR(큐싱) 차단, 세션 만료 후 장바구니 보존, POS 미연결 시 직원 호출 등 실제 운영 환경에서 생길 수 있는 예외 상황을 모두 사전 정의합니다.

📌 개발 범위 — 손님 화면 (customer.html) 전용
이 문서는 손님이 보는 화면에서 일어나는 모든 상태·분기·예외를 정의합니다. 사장님 관리 화면(index.html), POS 연동, 카카오 채널 관리는 별도 문서를 참고하세요.

SC_01

매장 · 셀프알림톡 · 선불

🍽️ 매장 / 📱 셀프알림톡 / 💳 선불 · happy-path
  1. QR 스캔 + 매장 진입 검증 (4 게이트)
  2. 메뉴 탐색 (로그인 불필요)
  3. 주문유형 = 매장, 서빙 = 셀프알림톡 자동 설정
  4. ★ 알림 시트 필수 노출 (건너뛰기 버튼 숨김)
  5. 약관 동의 (회원 2개 / 비회원 3개)
  6. 결제 (선불, 결제수단 6종)
  7. 호출번호 발급 + 카카오 알림톡
  8. 준비완료 → 카운터 수령
★ 준비완료 알림 없으면 손님이 수령 불가 → 알림 시트 스킵 절대 금지

📋 메뉴판 표시
  • 카테고리·메뉴·옵션·가격 렌더링
  • 품절 메뉴 비활성 표시
  • 재고 2개 이하 → INF_001 표시
  • 상단 대표 이미지(최대 3장) 자동 전환
  • 향후 디벨롭: 대표 이미지에 URL 삽입(탭 시 이동)
🛒 함께주문 (매장 전용)
  • 같은 테이블 손님끼리 장바구니 실시간 공유
  • 포장·예약 시 자동 차단
  • 장바구니에서 개별 결제 / 함께 결제 선택
  • 현재: 함께 결제 시 대표 1인이 전체 금액 결제 (결제 시 전체 잠금)
  • 향후: 1/N 분할 결제 디벨롭 필요
🔜 향후 과제 · 1/N 분할 결제
  • 현재는 한 명이 전체 금액을 결제
  • 향후 함께주문 참여 인원수로 금액을 나눠 각자 결제하는 1/N 분할 결제 기능으로 발전 필요
  • 분할 방식·정산 처리는 별도 기획·개발 검토
⏱ 세션 만료 (무활동 감시)

  • 만료 후에도 장바구니 일정 시간 보존

04

결제 5분기 — ★ 핵심 의사결정 트리

5단계 순서대로 결정 · 앞 분기가 뒤 분기를 제약
⚠️ 중요: 분기는 ①→②→③→④→⑤ 순서가 중요합니다. 앞 분기 결과가 뒤 분기 선택지를 제한합니다 (예: 주문유형 = 포장이면 서빙 분기 ②는 건너뜁니다).
📌 주문 유형은 진입 시점에 결정됩니다.
· 매장이 한 가지 유형만 운영하면 진입 시 자동으로 그 유형으로 시작합니다.
· 매장이 먹고가기 + 포장하기 등 2개 이상을 운영하면, 최초 화면에서 주문 방식을 선택하는 모달을 띄워 손님이 고릅니다. 선택지는 가로로 나란히(병렬) 배치합니다. (운영 유형은 사장님사이트에서 설정)
· 서빙 방식은 매장 설정으로 결정되며, 주문 유형이 바뀌면 담은 메뉴·쿠폰·연락처 설정은 초기화됩니다.
분기 ① 주문 유형 (3-way)
🍽️ 매장
  • → 분기 ② 서빙 방식으로 진행
  • 함께주문 활성화 가능
🥡 포장
  • → 분기 ② 건너뜀
  • → 분기 ③ 알림 시트 필수
  • 후불 선택 차단
  • 함께주문 차단
📅 예약
  • → 수령 시간 선택 단계 추가 (미선택 시 결제 버튼 비활성)
  • → 분기 ③ 알림 시트 필수
  • 함께주문 차단
분기 ② 서빙 방식 (매장 전용, 3-way)
🙋 테이블 서빙
  • → 분기 ③ 알림 시트 생략 (영수증 토글만)
  • 후불 가능 (유일)
  • 진행단계 3단계(접수→준비→완료)
  • 호출번호 미발급

  • 후불 차단
  • 호출번호 발급 (전광판)

  • → 분기 ③ 알림 시트 필수
  • 건너뛰기 버튼 완전 숨김
  • 후불 차단
  • 호출번호 발급 + 카톡 알림
분기 ③ 알림 시트 노출 여부 + 인증 수단 (★ 디폴트 = 휴대폰)
⚠️ 인증 수단 기본 정책 — 카카오·휴대폰을 둘 다 보여주지 않습니다.
· 디폴트는 휴대폰 번호 입력입니다. 평소에는 카카오 인증을 노출하지 않습니다.
· 사장님사이트에서 "카카오톡 인증 필수"를 켠 매장에서만 카카오 인증을 먼저 유도합니다.
· 즉, 카카오 인증 화면은 매장 설정에 따라 조건부로 나타납니다. (설정 OFF = 휴대폰 입력, 설정 ON = 카카오 우선)
📱 휴대폰 번호 입력 — 기본(디폴트)
  • 별도 설정이 없으면 항상 이 화면이 기본

  • 이미 동의한 번호는 체크박스 생략 · 동의 시 쿠폰 목록 표시
💛 카카오 인증 — 조건부(사장님 설정 ON)
  • 사장님사이트 "카카오톡 인증 필수" ON일 때만 노출
  • 이 경우 카카오 인증 화면을 먼저 보여줌
  • 로그인 성공 → 알림 시트(연락처 입력) 건너뜀
  • 카카오톡 없는 손님 → "카카오톡이 없어요." → 휴대폰/이메일 선택 화면
📧 이메일 입력 — 대체 수단
  • 카카오 대체 선택 화면 또는 직접 입력으로 진입
  • 이메일 형식 유효성 검사 → 이메일 확인 모달
  • 마케팅 수신동의 체크박스 제외 · 주문 알림 용도로만 사용
분기 ④ 약관 동의 (2-way)
💛 카카오 회원 — 2개
  • ✓ 이용약관 (필수)
  • ✓ 개인정보 처리방침 (필수)
  • 만 14세 이상 확인 생략 (가입 시 확인됨)
👤 비회원 — 3개
  • ✓ 이용약관 (필수)
  • ✓ 개인정보 처리방침 (필수)
  • ✓ 만 14세 이상 확인 (필수, 미동의 시 결제 차단)
분기 ⑤ 결제 모드 (2-way) — 선불 / 후불
💳 선불 (SC_01~SC_03, SC_05, SC_06 총 5개 시나리오)
  • 결제수단 6종 노출: 토스·카카오페이·네이버페이·신용카드·알리페이·위챗페이
  • 외부 결제(PG) 진행 · 중복 결제 방지 필요
  • 결제 성공 → 주문 접수 → 완료 화면
  • 결제 실패 → ERR_005 (장바구니 보존)
🧾 후불 (SC_04 단 1개 시나리오)
  • 결제수단 영역 전체 숨김
  • "주문하기 · ₩X원" 버튼만 노출
  • 결제 단계 없음 · 주문만 접수
  • 완료 후 "나갈 때 카운터 결제" 안내 카드 표시
  • 매장 + 서빙 방식만 가능 (셀프 + 후불 차단)
🎟️ 쿠폰 번호 직접 입력 (선택)
· 결제 페이지에서 쿠폰 번호를 직접 입력해 적용할 수 있습니다. (보유 쿠폰 목록 선택과 별개)
· 현장에서 받았거나 별도 종이 쿠폰 등으로 받은 쿠폰도 번호만 있으면 적용 가능합니다.
· 선택적 마케팅 수신동의 여부와 무관하게 사용할 수 있으며, 쿠폰 적용 자체가 선택 값입니다 (입력하지 않아도 결제 진행).

알림 시트 매트릭스 — 5케이스 필수/생략

연락처 수집 바텀 시트의 노출 조건
주문유형 서빙방식 시트 노출 이유 연락처 수단 스킵 허용
매장 서빙 (테이블) 생략 직원이 자리로 서빙 · 알림 불필요 영수증 카톡 수신 토글만 제공 —
매장 셀프 + 호명 생략 전광판 호명으로 충분 · 연락처 없이 가능 없음 —
매장 셀프 + 알림톡 ★ 필수 준비완료 알림 없으면 수령 불가 휴대폰(기본) · 이메일 / 카카오(사장님 설정 ON 시) 건너뛰기 버튼 숨김
포장 — ★ 필수 픽업 준비완료 알림 필요 휴대폰(기본) · 이메일 / 카카오(사장님 설정 ON 시) 건너뛰기 버튼 숨김
예약 — ★ 필수 준비·픽업 알림 필요 · 미수락 시 환불 안내 휴대폰(기본) · 이메일 / 카카오(사장님 설정 ON 시) 건너뛰기 버튼 숨김
알림 시트 내부 흐름 상세 (필수 케이스 한정)
⚠️ 디폴트는 휴대폰 번호 입력입니다. 카카오 인증은 사장님사이트 "카카오톡 인증 필수" 설정이 ON인 매장에서만 먼저 나타나고, 그 외에는 휴대폰 번호 입력이 기본 화면입니다.
📱 휴대폰 번호 입력 — 기본(디폴트)
  • 별도 설정이 없으면 항상 이 화면이 기본
  • 입력 완료 → "이 번호로 알림을 보내드려요" 확인 모달
  • 최초 입력 시 마케팅(쿠폰·이벤트) 수신동의 체크박스 표시 (선택)
  • 이미 동의한 번호는 체크박스 생략 · 동의 시 쿠폰 목록 표시
💛 카카오 인증 — 사장님 설정 ON일 때만
  • "카카오톡 인증 필수" ON 매장에서만 카카오 인증을 먼저 노출
  • 카카오 로그인 → 성공 시 알림 시트 즉시 종료, 주문 진행
  • "카카오톡이 없어요." 버튼 클릭 → 휴대폰/이메일 선택 화면
  • 선택 화면: 📱 휴대폰 / 📧 이메일 + "돌아가기"
📧 이메일 입력
  • 이메일 형식 유효성 검사 (실시간)
  • 입력 완료 → "이 이메일로 알림을 보내드려요" 확인 모달
  • 마케팅 수신동의 체크박스 제외 · 주문 알림 용도로만 사용
✓ 같은 기기 — 기존 로그인·연락처 정보 기억
같은 기기로 다시 주문하면 이전에 입력한 정보를 기억해 재입력을 생략합니다.
· 카카오톡 인증으로 로그인했던 경우 → 로그인 상태 유지, 다시 로그인하지 않아도 됨
· 휴대폰 번호를 입력했던 경우 → 해당 번호를 기억해 자동으로 채워 줌
· 손님이 번호를 바꾸려는 경우에만 직접 수정할 수 있게 함 (재입력 강제 아님)
✓ 마케팅(쿠폰·이벤트) 수신동의 — 휴대폰 입력 시 최초 1회
· 휴대폰 번호 입력 시 마케팅 수신동의를 받습니다. (향후 쿠폰을 휴대폰 번호로 발송하기 때문)
· 최초 1회만 받고, 이미 동의한 번호에는 이후 다시 띄우지 않습니다.
· 이메일 입력 시에는 마케팅 수신동의를 받지 않습니다. 이메일은 주문 알림 용도로만 사용합니다.

차단·블로킹 규칙 4종

위반 시 진행 불가 · UI에서 강제 비활성
🚫 포장·예약 = 항상 선불

포장 또는 예약 선택 시 후불 결제 옵션을 UI에서 숨깁니다. 후불은 매장 + 서빙 조합에서만 노출됩니다.

🚫 예약 = 수령시간 미선택 시 결제 차단

예약 주문에서 수령 시간(시·분)을 선택하지 않으면 결제 버튼이 비활성 상태를 유지합니다. 결제 시도 시 "수령 시간을 선택해 주세요" 안내를 표시합니다.

🚫 포장·예약 = 함께주문 차단

실시간 공동 장바구니(함께주문)는 매장 내 식사 전용입니다. 포장·예약 선택 시 함께주문 버튼을 비활성화하고, 이미 연결된 경우 연결을 해제합니다.

🚫 셀프 + 알림톡 = 건너뛰기 금지

서빙 방식이 셀프알림톡이거나 포장·예약인 경우 알림 시트의 "건너뛰기" 버튼을 숨겨 연락처 수집을 강제합니다. 알림 없이 주문이 완료되면 수령이 불가능합니다.
