# 메뉴 관리

`SPEC.md` §3 화면 인덱스의 상세 명세. **메뉴 관리** 영역의 화면·필드·노출값(손님 노출 기준)·동작·검증을 개발팀이 그대로 구현할 수 있도록 정리합니다.

이 문서의 핵심은 **§1.2 메뉴 필드 상세 명세**와 **§1.3 노출값 기준**입니다. 사장님이 어드민에서 입력·관리하는 값 중 **무엇이 손님 주문 화면에 어떻게 노출되는지**를 필드 단위로 확정합니다.

데이터의 DB 논리 모델은 `DATA_MODEL.md` §2(메뉴/옵션/카테고리)를 기준으로 하며, 본 문서는 **화면·기능·노출 관점**을 다룹니다. 두 문서 사이에 보완이 필요한 항목은 §7 데이터 모델 보완 포인트에 모았습니다.

---

## 0. 영역 개요

### 0.1 구성 (좌측 상단 탭)

메뉴 관리는 다음 5개 하위 화면으로 구성됩니다. (프로토타입 `menuNavTabs()` 기준)

| 탭 | 경로 키 | 다루는 데이터 | 본 문서 절 |
|---|---|---|---|
| 🍽️ 메뉴판 | `menu` | 단품 메뉴 (`MENUS`) | §1 |
| 🧩 옵션 선택지 | `option` | 옵션 그룹·항목 (`OPT_GROUPS`) | §2 |
| 🗂️ 카테고리 | `category` | 손님 메뉴판 분류 (`CAT_MENU`) | §3 |
| 🎁 세트 | `set` | 세트 메뉴 (`SETS`) | §4 |
| 🌿 원산지 | `origin` | 원산지 표기 (`STORE_ORIGIN`) | §5 |

> 다국어(번역)는 `SPEC.md` 인덱스와 `DATA_MODEL.md` §2.11에 정의되어 있으며, 현재 프로토타입에서는 카테고리 화면의 **언어 미리보기 토글**로 손님 화면 다국어 노출을 확인합니다. 메뉴·옵션·카테고리·세트의 `name`/`desc`가 번역 대상입니다. (§6)

각 화면 상단 breadcrumb : `메뉴 관리 › {화면명}`.

### 0.2 핵심 개념 — 카테고리 vs 분류 태그 (반드시 구분)

| 구분 | 필드 | 목적 | 손님 노출 |
|---|---|---|---|
| **카테고리** | `MENUS.cat` (→ `CAT_MENU`) | 손님 메뉴판의 분류 탭 (커피 / 음료 / 디저트) | 🟢 **노출** |
| **분류 태그** | `MENUS.catL2` / `catL3` / `tag3` | 사장님이 어드민에서 **메뉴를 검색·필터**할 때 쓰는 자유 키워드 (대/중/소 3단) | ⚪ **미노출** |

> 메뉴가 수십~수백 개로 늘어나면 사장님이 어드민에서 원하는 메뉴를 빨리 찾기 위해 분류 태그(대/중/소)를 씁니다. 이 값은 **손님 화면에 절대 노출되지 않습니다.** 손님에게 보이는 분류는 오직 `cat`(카테고리)뿐입니다.

### 0.3 노출값 표기 범례

본 문서 전반의 "손님 노출" 열 표기 :

| 표기 | 의미 |
|---|---|
| 🟢 노출 | 손님 주문 화면(메뉴판·메뉴 상세)에 직접 표시 |
| ⚪ 미노출 | 사장님 어드민에서만 사용. 손님 화면에 표시되지 않음 |
| 🔵 조건부 | 특정 조건에서만 노출되거나, 조건에 따라 노출 형태가 달라짐 |

## 1. 메뉴판 (`menu`)

### 1.1 화면 목적

단품 메뉴를 등록·수정하고, 카테고리·옵션·판매 시간·할인을 연결합니다. 손님 메뉴판에 보이는 메뉴 목록의 원본입니다.

페이지 부제 : "단품을 등록하고, 옵션 그룹을 연결해요. 여러 개를 한 번에 올릴 수 있어요."

### 1.2 메뉴 필드 상세 명세

한 메뉴(`MENUS[i]`) 객체의 전체 필드입니다. 기본값은 `ensureMenuExtras(m)`가 최초 접근 시 채웁니다.

#### 1.2.1 손님에게 노출되는 값 (핵심)

| 필드 | 화면 라벨 | 타입 | 필수 | 기본값 | 검증 | 손님 노출 |
|---|---|---|---|---|---|---|
| `name` | 메뉴 이름 / 메뉴명 | string(1~30) | ✅ | — | 빈 값 불가. 같은 이름 존재 시 저장 전 확인(차단 X) | 🟢 메뉴판 메뉴명 |
| `price` | 매장 판매가 | int(원) | ✅ | `0` | 0 이상 정수, 1억 미만 | 🟢 매장 식사 시 가격 |
| `takeoutPrice` | 포장가 | int \| `null` | ⬜ | `null` | 0 이상. 매장가보다 비싸면 저장 전 확인 | 🔵 포장 주문 시 적용. `null`이면 매장가와 동일하게 노출. 단 주문 방식에서 **포장 할인 OFF면 미적용** |
| `cat` | 카테고리 | string (`CAT_MENU.name`) | ✅ | (일괄입력 시 '미지정' 허용) | `CAT_MENU`에 존재하는 이름 | 🟢 메뉴판 분류 탭 |
| `desc` | 설명 | string(~200) | ⬜ | `''` | — | 🟢 메뉴명 아래 작은 글씨. 비우면 미표시 |
| `img` | 메뉴 사진 | string (이모지 \| `data:` \| `http(s)` URL) | ⬜ | `''` | 이미지 파일 JPG·PNG, 5MB 이하 | 🔵 노출. **비우면 손님 화면에 사진 영역 자체가 표시되지 않음** |
| `maxQty` | 1회 주문 최대 수량 | int \| `null` | ⬜ | `null`(무제한) | 1 이상 정수. 0·빈값이면 `null` 처리 | 🔵 담기 수량 상한. 어드민 목록엔 '최대 N개' 배지 |
| `opt` | 연결된 옵션 | number[] (`OPT_GROUPS.id`) | ⬜ | `[]` | 존재하는 옵션 그룹 id | 🟢 메뉴 상세의 옵션 선택지 (§2) |
| (배열 순서) | 순서 | 정렬(드래그) | — | 등록 순 | — | 🟢 **손님 메뉴판도 여기서 정한 순서 그대로 노출** |

#### 1.2.2 사장님 내부 전용 값 (손님 미노출)

| 필드 | 화면 라벨 | 타입 | 필수 | 기본값 | 검증 | 손님 노출 |
|---|---|---|---|---|---|---|
| `id` | — | number | (자동) | 자동 부여 | PK | ⚪ 내부 식별자 |
| `code` | 상품코드 | string(6) | ✅(자동생성) | `genCode('M')` | 영문 1자(M) + 숫자 5자리, **매장 내 유니크**(메뉴·옵션·세트 공통 풀) | ⚪ 어드민·정산·POS 매칭용 |
| `catL2` | 분류 ① (대분류) | string | ⬜ | `''` | — | ⚪ 어드민 검색·필터 전용 |
| `catL3` | 분류 ② (중분류) | string | ⬜ | `''` | — | ⚪ 동일 |
| `tag3` | 분류 ③ (소분류) | string | ⬜ | `''` | — | ⚪ 동일 |

> `code`는 ⚙️ 버튼으로 자동 생성하거나 사장님이 직접 6자로 지정할 수 있습니다. 생성 규칙은 §1.6 검증 참조.
> 분류 태그(대/중/소)는 편집 모달에서 **자동완성(datalist)** 으로 기존에 입력한 값을 제안합니다. 손님 화면에는 노출되지 않습니다.

#### 1.2.3 운영·상태 파생 값 (재고·품절·시간·할인 연동)

| 필드 | 화면 라벨 | 타입 | 기본값 | 의미 / 손님 노출 |
|---|---|---|---|---|
| `status` | 판매 상태 | enum `판매중` \| `품절` \| `숨김` | `판매중` | 🔵 `effectiveStatus`로 손님 노출 제어 (§1.4) |
| `manualSoldOut` | (오늘 품절 토글) | bool | `false` | 🔵 수동 품절. **새 영업일 시작 시 자동 해제** |
| `soldOutKind` | (품절 종류) | enum `temp` \| `full` | `temp` | `temp`=일시 품절(준비 후 재판매), `full`=완전 품절(오늘 판매 불가). `manualSoldOut=true`일 때만 의미 |
| `stock` / `sold` | (오늘의 준비량 연동) | int \| `null` / int | `null` / `0` | 🔵 `오늘의 준비량`(`MENU_STOCK`)과 연동. 잔여(=`stock−sold`)가 0이면 자동 품절. `null`이면 수량 무제한 (`spec/03-stock.md`) |
| `schedule` | 판매 시간 | `{enabled, days[], start, end}` | `{enabled:false, days:[월~일], 09:00~22:00}` | 🔵 비활성이면 '항상' 판매. 활성 시 시간 외에는 손님 화면에서 회색 + '판매 시간 아니에요' |
| `discount` | 해피아워 할인 | `{enabled, days[], start, end, rate}` | `{enabled:false, days:[월~금], 14:00~17:00, rate:20}` | 🔵 활성 + 시간 충족 시 손님에게 할인가 노출. `rate`는 % (1~100) |

`schedule.days` / `discount.days`는 요일 정수 배열(0=일 … 6=토).

### 1.3 노출값 기준 (사장님 어드민 ↔ 손님 주문 화면)

> 같은 메뉴 데이터가 두 화면에서 어떻게 보이는지의 확정 기준입니다. 개발 시 손님 앱(주문 화면)과 사장님 어드민의 렌더링 분기에 사용하세요.

| 값 | 사장님 어드민 | 손님 주문 화면 |
|---|---|---|
| 메뉴명 `name` | 목록·모달 표시 | 🟢 메뉴 카드 제목 |
| 매장 판매가 `price` | 표시 | 🟢 매장 식사 시 가격 |
| 포장가 `takeoutPrice` | 표시(비면 '매장가 동일') | 🔵 포장 채널에서만 적용. `null`→매장가. 포장 할인 OFF면 매장가로 노출 |
| 카테고리 `cat` | 분류 탭·필터 | 🟢 메뉴판 상단 분류 탭 |
| 설명 `desc` | 표시 | 🟢 메뉴명 아래 보조 텍스트(있을 때만) |
| 사진 `img` | 36px 썸네일 | 🔵 있을 때만 사진 노출, 없으면 사진 영역 생략 |
| 1회 최대 수량 `maxQty` | '최대 N개' 배지 | 🔵 담기 수량 상한 적용 |
| 연결 옵션 `opt` | 칩 목록 | 🟢 메뉴 상세 옵션 선택지 |
| 메뉴 순서 | 드래그 정렬 | 🟢 메뉴판 노출 순서와 동일 |
| 분류 태그 `catL2/catL3/tag3` | 드롭다운 필터·검색 | ⚪ **노출 안 함** |
| 상품코드 `code` | 표시 | ⚪ 노출 안 함 |
| 판매 상태 `status` | pill 표시·필터 | 🔵 `숨김`→비표시 / `품절`→회색·선택 불가 (§1.4) |
| 오늘 준비량(잔여) | '남은/준비' + 진행바 | 🔵 0이면 '품절'로 노출, 그 외엔 수치 비노출 |
| 판매 시간 `schedule` | 표시 | 🔵 시간 외 회색 + '판매 시간 아니에요' |
| 해피아워 `discount` | '🔥 N%↓' 표시 | 🔵 적용 시간대에 할인가 노출 |

### 1.4 메뉴 효과적 상태(`effectiveStatus`)와 손님 노출 규칙

여러 상태 입력을 하나의 "지금 손님에게 어떻게 보이는가"로 합성합니다. 우선순위(위 → 아래, 먼저 맞는 것 적용) :

```
1. manualSoldOut = true            → 수동 품절  (soldOutKind: temp=일시 / full=완전)
2. stock != null && sold >= stock  → 자동 품절  (오늘의 준비량 소진)
3. status = '숨김'                 → 숨김
4. status = '품절'                 → 품절       (레거시; §7 참조)
5. (그 외)                         → 판매중
```

손님 화면 노출 규칙 (`SPEC.md` §2.4와 동일) :

| 효과적 상태 | 손님 화면 |
|---|---|
| 판매중 + 판매 시간 충족 | 정상 노출, 선택 가능 |
| 판매 시간 미충족 | 회색 + '판매 시간 아니에요' (선택 불가) |
| 수동/자동 품절 | 회색 + '품절' 라벨 (선택 불가) |
| 숨김 | **목록에서 제외** (아예 표시 안 함) |

어드민 pill 매핑 : 판매중=`done`, 품절·완전/일시 품절=`partial`/`cancel`, 숨김=`hold`.

### 1.5 화면 구성 & 동작

#### 1.5.1 새 메뉴 입력 (일괄 등록 표)

접었다 펼치는 영역(`✏️ 새 메뉴 입력`). 한 번에 여러 메뉴를 표로 입력합니다. **저장 전에는 손님 화면에 보이지 않습니다.**

표 컬럼 : `#` · 상품코드(자동) · 메뉴 이름(필수) · 매장 판매가(필수) · 포장가(선택) · 카테고리 · 설명 · 사진 · 상태 · 행 삭제(✕).

- 새 행 기본값 : `{ code:'', name:'', price:0, takeoutPrice:null, cat:'', desc:'', img:'', status:'판매중' }`
- 일괄 입력 표의 상태 옵션은 **판매중 / 숨김** 2가지만(품절은 등록 단계에서 선택 불가 — §1.4 참조).
- 상품코드 ⚙️ : `genCode('M')` 자동 생성. 사진 자리 클릭 : 샘플 이모지 순환(`SAMPLE_ICONS`).
- 액션 : `+ 메뉴 행 추가` / `행 비우기` / **저장**(`saveMenuBulk`).
- **엑셀로 한 번에 올리기** : `📤` → JSON 파일 import(`importMenuJSON`). 형식은 §1.5.6.

#### 1.5.2 등록된 메뉴 표

표 컬럼(좌→우) : 선택(체크박스, 전체선택) · 순서(≡ 드래그 + ⓘ 도움말) · 사진 · 상품코드 · 메뉴명(+ '최대 N개' 배지 + 설명) · 카테고리 · 매장 판매가(+ 해피아워) · 포장가(없으면 '매장가 동일') · 오늘 준비량(남은/준비 + 진행바) · 판매 시간 · 연결된 옵션 · 상태(pill) · 추가 설정 ▼.

- **행 클릭** → 메뉴 정보 수정 모달(§1.5.3).
- **순서 변경** : ≡ 손잡이 드래그 앤 드롭. 손님 메뉴판 순서와 동일. ⓘ 클릭 시 안내 팝업.
- **오늘 준비량 셀** 클릭 → `오늘의 준비량` 화면 이동. 잔여 0이면 '🔴 오늘 완료', 임계값 이하면 '⚠️ 곧 품절'.

#### 1.5.3 메뉴 정보 수정 모달 (`renderMenuEditModal`)

입력 순서 : 사진 → 메뉴 이름(필수) → 매장 판매가(필수) / 포장가(선택) → 카테고리(필수, `+ 새 카테고리`) → 🏷️ 분류 태그 ①②③(선택·손님 미노출) → 판매 상태 → 설명 → 1회 주문 최대 수량.

- 포장 할인 OFF 시 포장가 입력란 위에 경고 배너 + '설정 →'(주문 방식 이동).
- 오늘의 준비량 연동 시 현재 잔여 요약 + '준비량 변경 →'.
- 저장(`saveMenuEdit`) 검증은 §1.6.

#### 1.5.4 추가 설정 ▼ (행별 메뉴)

| 항목 | 동작 |
|---|---|
| ✏️ 메뉴 정보·가격 수정 | 편집 모달 (§1.5.3) |
| 🧩 옵션 그룹 연결 | `openOptLinkModal` — 이 메뉴에 옵션 그룹 연결/해제 (`opt` 배열) |
| 🕒 판매 시간 | `openScheduleModal('menu', id)` — `schedule` 설정 |
| 🔥 해피아워 할인 | `openDiscountModal('menu', id)` — `discount` 설정 |
| 📋 복제하기 | `duplicateMenu` — 새 코드로 복제 |
| ✕ 삭제 | `deleteMenu` — 연결 데이터 정리 후 삭제(§1.5.5) |

#### 1.5.5 일괄 작업 (체크박스 선택 시)

| 버튼 | 동작 |
|---|---|
| 📋 복제 | `duplicateSelectedMenus` |
| 💰 가격 조정 | `openBulkPriceModal` — 선택 메뉴 가격 일괄 변경 |
| 🗂️ 카테고리 이동 | `openBulkCatModal` — 선택 메뉴 `cat` 일괄 변경 |
| ✕ 삭제 | `bulkDeleteMenus` — 확인 후 삭제 |

**삭제 시 연결 데이터 정리** : 메뉴 삭제 시 `오늘의 준비량`(`MENU_STOCK`), 세트 후보(`SETS…candidates`), 옵션 연결(`OPT_GROUPS.links`)에서 해당 메뉴 참조를 함께 정리합니다. 확인 문구 : "'{메뉴명}' 메뉴를 삭제할까요? 오늘의 준비량·세트 후보·옵션 연결도 함께 정리돼요."

#### 1.5.6 가져오기 / 내보내기 / 초기화

- **엑셀 다운로드** `exportMenuJSON` → `{ menus, optGroups, catMenu, sets, exportedAt }` JSON. 파일명 `qrorder-menu-YYYY-MM-DD.json`.
- **엑셀로 한 번에 올리기** `importMenuJSON` → 같은 구조 JSON 업로드(확인 후 **전체 덮어쓰기**).
- **모두 지우기** `resetMenuAll` → 메뉴·옵션·카테고리·세트 전체 삭제(되돌릴 수 없음, 확인 필수).

### 1.6 검증 규칙

| 필드 | 규칙 | 위반 시 |
|---|---|---|
| 메뉴 이름 | 1~30자, 빈 값 불가 | 저장 차단 + "메뉴 이름을 입력해 주세요." |

## 2. 옵션 선택지 (`option`)

### 2.1 화면 목적

메뉴에 붙는 옵션 그룹(예: 샷 추가, 사이즈, 얼음·온도)과 그 항목을 관리합니다. 한 그룹을 여러 메뉴에 연결할 수 있습니다.

### 2.2 옵션 그룹 / 항목 필드 명세

| 필드 | 화면 라벨 | 타입 | 필수 | 기본값 | 손님 노출 |
|---|---|---|---|---|---|
| `id` | — | number | 자동 | 자동 | ⚪ |
| `code` | 상품코드 | string(6) | ✅(자동) | `genCode('O')` | ⚪ |
| `name` | 옵션 그룹명 | string(~30) | ✅ | — | 🟢 옵션 그룹 제목 |
| `mode` | 선택 방식 | enum `1개 선택`(single) \| `중복 선택`(multi) | ✅ | `1개 선택` | 🔵 single=라디오, multi=체크박스 |
| `required` | 필수 선택 | bool | ✅ | `false` | 🔵 true면 손님이 반드시 1개 이상 선택 |
| `items[]` | 옵션 항목 | `{ name, price }[]` | ✅(1개↑) | — | 🟢 선택지 목록 |
| `items[].name` | 항목명 | string(~30) | ✅ | — | 🟢 |
| `items[].price` | 추가 금액 | int(원) | ✅ | `0` | 🔵 **음수 허용**(감액). `0`이면 추가금 표시 안 함 |
| `links[]` | 연결된 메뉴 | number[] (`MENUS.id`) | ⬜ | `[]` | (연결 메뉴의 상세에 노출) |

옵션 항목의 오늘 품절(아래)은 `ensureOptExtras(it)`가 기본값을 채웁니다.

| 필드 | 타입 | 기본값 | 의미 |
|---|---|---|---|
| `items[].manualSoldOut` | bool | `false` | 옵션 항목 단위 수동 품절. 🔵 품절 항목은 손님이 선택 불가 |
| `items[].soldOutKind` | enum `temp`\|`full` | `temp` | 일시/완전 품절 구분 (`오늘의 준비량` §3과 동일 의미) |

> 옵션 항목도 메뉴처럼 **오늘의 준비량에서 수량 관리·품절 처리**할 수 있습니다. (`OPT_STOCK`, `spec/03-stock.md`)

### 2.3 동작 / 노출 규칙

- 한 옵션 그룹을 여러 메뉴에 연결(`links`) — N:M. 메뉴 쪽에서는 `MENUS.opt`로 참조.
- `required=true` + `mode=1개 선택` : 기본 항목이 미리 선택된 상태로 노출.
- 항목 `price>0` → '+600원'처럼 추가금 노출, `price=0` → 추가금 없음, `price<0` → 감액.
- 품절 항목 : 회색 + 선택 불가. 그룹의 모든 항목 품절 시 그룹째 비활성.

## 3. 카테고리 (`category`)

### 3.1 화면 목적

손님 메뉴판 상단의 분류(탭/구획)를 관리합니다. 메뉴의 `cat` 값이 여기를 참조합니다.

### 3.2 필드 명세

| 필드 | 화면 라벨 | 타입 | 필수 | 기본값 | 손님 노출 |
|---|---|---|---|---|---|
| `id` | — | number | 자동 | 자동 | ⚪ |
| `name` | 카테고리명 | string(~30) | ✅ | — | 🟢 메뉴판 분류 탭 |
| `order` | 순서 | int / 드래그 정렬 | ✅ | 등록 순 | 🟢 분류 노출 순서 |
| `displayMode` | 노출 모드 | enum `grid` \| `list` | — | `grid` | 🔵 손님 메뉴판에서 그리드/리스트 형태 |
| `paused` | 운영 중지 | bool | — | `false` | 🔵 true면 **해당 카테고리·소속 메뉴가 손님 화면에서 빠짐** |

`displayMode`/`paused`는 `ensureCatExtras(c)`가 기본값을 채웁니다.

### 3.3 동작 / 노출 규칙

- 카테고리 행에서 노출 모드(그리드/리스트) 토글, 운영 중지(`paused`) 토글, 판매 시간(카테고리 단위 `openCategoryScheduleModal`) 설정.
- 카테고리 삭제 시 소속 메뉴의 `cat` 처리(미지정 이동 등)는 개발 시 정책 확정 필요 — §7.
- 자동/수동 품절·운영 중지된 카테고리·판매 시간 외 메뉴는 손님 화면에서 자동으로 빠지거나 회색 처리됩니다. (안내 문구: "자동 품절·수동 품절·운영 중지된 카테고리·판매 시간 외 메뉴는 자동으로 빠지거나 회색 처리돼요.")

## 4. 세트 (`set`)

### 4.1 화면 목적

여러 자리(slot)로 구성된 세트 메뉴를 관리합니다. 손님은 자리마다 후보 중 하나를 골라 자기 조합을 만듭니다.

### 4.2 필드 명세

| 필드 | 화면 라벨 | 타입 | 필수 | 기본값 | 손님 노출 |
|---|---|---|---|---|---|
| `id` | — | number | 자동 | 자동 | ⚪ |
| `code` | 상품코드 | string(6) | ✅(자동) | `genCode('S')` | ⚪ |
| `name` | 세트명 | string(~30) | ✅ | — | 🟢 |
| `basePrice` | 기본 가격 | int(원) | ✅ | — | 🔵 최저 조합 가격의 기준 |
| `desc` | 설명 | string(~200) | ⬜ | `''` | 🟢 |
| `enabled` | 판매 여부 | bool | ✅ | `true` | 🔵 false면 손님 미노출 |
| `slots[]` | 구성 자리 | object[] | ✅(1개↑) | — | 🟢 자리별 선택 UI |
| `slots[].name` | 자리명 | string | ✅ | — | 🟢 (예: '음료 선택') |
| `slots[].selectionType` | 선택 방식 | enum `single` | ✅ | `single` | 🔵 자리당 1개 선택 |
| `slots[].candidates[]` | 후보 메뉴 | object[] | ✅(1개↑) | — | 🟢 선택지 |
| `…candidates[].code` | 메뉴 코드 | string (`MENUS.code`) | ✅ | — | (메뉴 참조) |
| `…candidates[].name` | 메뉴명(표시) | string | ✅ | — | 🟢 |
| `…candidates[].extraPrice` | 추가 금액 | int(원) | ⬜ | `0` | 🔵 **음수 허용**. 이 후보 선택 시 가감 |
| `…candidates[].isDefault` | 기본 선택 | bool | ⬜ | `false` | 🔵 자리의 기본 선택 후보 |

세트도 `schedule`/`discount`를 가질 수 있습니다(`ensureSetExtras`, 메뉴와 동일 구조).

### 4.3 가격 계산 (`setPriceRange`)

```
min = basePrice + Σ(자리별 min(extraPrice))
max = basePrice + Σ(자리별 max(extraPrice))
```

- 손님 화면 : 자리마다 `isDefault` 후보가 기본 선택된 상태로 시작, 합계 실시간 표시.
- 어드민 목록 : `min`~`max` 범위 노출(단일 조합이면 단일 가격).

## 7. 데이터 모델 보완 포인트 (개발 인계용)

프로토타입에서 쓰지만 `DATA_MODEL.md`에 아직 반영되지 않았거나, 양쪽이 어긋나 **개발 전 확정이 필요한** 항목입니다.

| # | 항목 | 프로토타입 현황 | 권장 처리 |
|---|---|---|---|
| 1 | **판매 상태 값** | `status` 3값(`판매중`/`품절`/`숨김`) 사용 | 확정 모델은 `available`/`hidden` 2값 + 품절은 `manualSoldOut`·재고로 파생. 레거시 `품절`은 `manualSoldOut=true`로 마이그레이션 (`SPEC.md` §2.3) |
| 2 | **분류 태그** `catL2`/`catL3`/`tag3` | 메뉴에 owner 전용 3단 태그 보유, 손님 미노출 | `DATA_MODEL.md` §2.2 Menu에 추가하거나 별도 `MenuTag`(대/중/소) 엔터티로 정규화 |
| 3 | **품절 종류** `soldOutKind`(temp/full) | 메뉴·옵션 항목 모두 보유 | Menu/OptionItem에 `sold_out_kind` enum 추가 |
| 4 | **옵션 항목 재고/품절** | `OPT_STOCK` + 항목 `manualSoldOut` | OptionItem 단위 재고·품절 모델 추가 (현 `DATA_MODEL.md` StockItem은 메뉴 단위만) |
| 5 | **카테고리 노출 옵션** `displayMode`/`paused` | 보유 | MenuCategory에 `display_mode`(grid/list)·`paused` 추가 |
| 6 | **세트 구조** | slot + candidate + `extraPrice` + `isDefault` 모델 | `DATA_MODEL.md` §2.8/2.9의 단순 `SetMenuItem(menu_id, quantity)` 모델을 **자리/후보/추가금 모델로 갱신** |
| 7 | **원산지** | 자유 텍스트(`STORE_ORIGIN.text`) | `Origin(ingredient, origin)` 구조화 모델로 전환 + 마이그레이션 |
| 8 | **상품코드 형식** | 접두 영문 1자(M/O/S) + 숫자 5자리 = 6자 | 공통 검증 "영문 대문자+숫자 6자"와 일치. 접두 규칙(M=메뉴 등) 명문화 권장 |
