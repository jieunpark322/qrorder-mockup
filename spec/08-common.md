# 공통 UX 패턴

`SPEC.md` §1 공통 기준의 보충 문서. 모든 화면에 적용되는 인터랙션·시각·접근성 표준입니다.

---

## 1. 알림 / 토스트

### 1.1 종류

| 종류 | 색 | 자동 닫힘 | 닫기 버튼 |
|---|---|---|---|
| success | 초록 (`#1eb18d`) | 3초 후 | ✕ |
| info | 파랑 (`#3182f6`) | 4초 후 | ✕ |
| warning | 노랑 (`#f59e0b`) | 5초 후 | ✕ |
| error | 빨강 (`#f04452`) | 닫힘 없음 (사용자가 직접 닫음) | ✕ |

### 1.2 위치 / 동작

- 화면 우상단, 헤더 아래 16px 마진
- 여러 개 발생 시 최신이 위로 쌓이며 최대 3개까지 동시 표시. 초과분은 큐 대기
- 마우스 오버 시 자동 닫힘 타이머 정지
- 클릭 시 관련 화면으로 이동(옵션). 예 : "신규 주문이 들어왔어요" → 주문 내역 화면

### 1.3 사용 시점

- 저장 / 삭제 / 일괄 처리 등 명시적 액션 결과는 토스트로 표시
- 단순 조회 화면에서는 토스트 미사용

---

## 2. 로딩 상태

### 2.1 종류

| 패턴 | 사용 상황 |
|---|---|
| 스켈레톤 | 전체 화면 로드 시. 표·카드 형태에 맞게 회색 박스 |
| 인라인 스피너 | 버튼 클릭 후 응답 대기 (버튼 텍스트 → "처리 중...") |
| 글로벌 오버레이 | 결제 / 정산 등 시간 걸리는 트랜잭션. 화면 전체에 반투명 + 중앙 스피너 + 상태 메시지 |

### 2.2 동작

- 200ms 이하로 끝나는 요청은 로딩 표시 생략 (깜빡임 방지)
- 5초 이상 응답 없을 시 "시간이 좀 걸려요. 잠시만 더 기다려 주세요" 안내 추가
- 30초 이상 응답 없을 시 자동 취소 + 에러 토스트

---

## 3. 빈 상태 (Empty State)

### 3.1 공통 구조

```
[일러스트 / 큰 이모지]   (40~48px)
[제목 한 줄]              (15px, bold)
[설명 한두 줄]            (13px, ink-3)
[액션 버튼]               (선택)
```

배경 : `var(--bg-2)` (`#f2f4f6`), border-radius 14px, padding 48px 20px.

### 3.2 톤

- 친근하고 행동을 유도하는 문장 (예: "아직 등록된 메뉴가 없어요" + "+ 새 메뉴 추가" 버튼)
- 부정 표현 회피 ("없어요" 대신 가능형 "추가해 주세요" 우선)
- 절대 사용 금지 : 경고·주의 등 공포 표현

### 3.3 화면별 빈 상태 예시

각 spec 파일의 "빈 상태" 섹션 참조. mockup에 정의된 문구를 그대로 사용합니다.

## 4. 에러 상태

### 4.1 종류

| 종류 | 처리 |
|---|---|
| 네트워크 오류 (오프라인) | 화면 상단에 노란 배너 "네트워크 연결을 확인해 주세요". 자동 재시도 3회 |
| 서버 오류 (5xx) | 에러 토스트 + "다시 시도" 버튼. 동일 요청 5회 실패 시 본사 자동 알림 |
| 검증 오류 (4xx) | 입력 필드 옆에 빨간 라인 + 메시지. 토스트 미사용 |
| 권한 부족 (403) | 모달 "사장님 권한이 필요해요. 가맹점주 본인 계정으로 로그인해 주세요" |
| 세션 만료 (401) | 즉시 로그인 화면으로 리다이렉트 + "다시 로그인이 필요해요" 안내 |
| 데이터 없음 (404) | 빈 상태로 처리 (에러 아님) |

### 4.2 메시지 톤

- "오류가 발생했습니다" 같은 시스템적 표현 회피
- "잠시 후 다시 시도해 주세요" / "본사 고객센터로 문의해 주세요" 등 다음 행동 유도

## 5. 폼 / 검증

### 5.1 입력 필드

- label 위치 : 필드 위
- 필수 표시 : 라벨 우측 빨간 `*`
- placeholder : 예시 값 또는 입력 안내 ("예) 아메리카노")
- 보조 설명 : 필드 아래 작은 회색 글씨

### 5.2 검증

- 실시간 검증 (입력 변경 시 즉시 표시)
- 검증 실패 시 :
  - 필드 테두리 빨강
  - 필드 아래 빨간 메시지
  - 저장 버튼은 폼 전체 검증 실패 시 비활성
- 검증 통과 시 별도 표시 안 함 (초록 체크 등 미사용)

### 5.3 공통 검증 규칙

| 필드 유형 | 규칙 |
|---|---|
| 메뉴명 | 1~30자, 빈 값 불가, 같은 카테고리 내 중복 차단 |
| 가격 | 0 초과 정수, 1억 미만 |
| 상품코드 | 영문 대문자·숫자 6자리, 매장 내 중복 차단 (메뉴·옵션·세트 동일 규칙) |
| 전화번호 | 숫자만, 9~11자리 |
| 이메일 | RFC 5322 간소 검증 |
| 시간 | `HH:mm` 24시간제 |

### 5.4 저장 동작

- 저장 버튼은 dirty 상태일 때만 활성
- 저장 성공 시 success 토스트 + dirty 해제
- 저장 실패 시 error 토스트 + 폼 유지 (입력 값 손실 방지)

## 6. 모달 / 팝업

### 6.1 구조

```
[제목]              [✕ 닫기]
─────────────────────
[본문]
─────────────────────
[취소]    [확정]
```

- max-width : 360 ~ 640px (내용에 따라)
- 배경 : `rgba(0,0,0,0.5)` 반투명 + blur
- 모달 표시 중 body 스크롤 잠금

### 6.2 동작

- ESC 키로 닫기 (단, dirty 상태면 확인 prompt)
- 배경 클릭으로 닫기 (위와 동일)
- 모달 안에서 다른 모달 호출 가능 (스택)
- 확정 버튼은 폼 검증 통과 시 활성

### 6.3 종류

| 종류 | 사용 시점 |
|---|---|
| confirm | 단순 확인 (예: 삭제 시 "정말 삭제할까요?") |
| form | 데이터 입력·수정 (예: 메뉴 정보 모달) |
| info | 정보 표시 (예: 매출 집계 기준 안내) |
| change-warning | 영향 큰 변경 경고 (예: 주문 채널 변경) |

## 7. 접근성

### 7.1 시각

- 최소 텍스트 크기 : 12px (보조 설명만, 본문은 13px 이상)
- 본문 대비 : 4.5 : 1 이상 (WCAG AA)
- 색만으로 의미 전달 금지. 아이콘 / 텍스트 라벨 병행
- 50~60대 사장님 사용을 고려해 주요 액션 버튼은 14px 이상, 터치 영역 44x44px 이상

### 7.2 키보드

- 모든 인터랙티브 요소는 Tab 으로 접근 가능
- 모달 열림 시 첫 입력 필드로 자동 포커스
- 모달 닫힘 시 호출 위치로 포커스 복귀

### 7.3 스크린리더

- 아이콘 버튼은 `aria-label` 필수
- 토스트는 `role="status"` 또는 `role="alert"` (에러는 alert)
- 표 헤더는 `<th scope="col">`

### 7.4 모션

- 애니메이션은 200ms 이하 (긴 trans 금지)
- `prefers-reduced-motion` 사용자에게는 transition 비활성

## 8. 네비게이션

### 8.1 좌측 사이드바

- 항상 표시 (모바일에서는 햄버거 토글)
- 그룹 헤더 클릭으로 접기/펼치기
- 현재 선택된 항목 강조 (파란 배경)
- 권한 없는 항목 : 회색 + 자물쇠 아이콘. 클릭 시 권한 안내 모달

### 8.2 상단 헤더

- 좌측 : 매장명 + 사장님 이름
- 우측 : 알림 종 + 설정 + 로그아웃
- 알림 종 : 미확인 건수 빨간 배지. 클릭 시 알림 패널

### 8.3 빵 부스러기 (Breadcrumb)

- 모든 화면 상단에 표시
- 형식 : `상위 그룹 › 하위 그룹 › 현재 화면`
- 각 단계 클릭 시 해당 화면으로 이동

## 9. 색상 / 디자인 토큰

`index.html`의 CSS 변수와 동일. 디자인 팀에서 별도 시스템(Figma 등)으로 관리.

| 토큰 | 값 | 용도 |
|---|---|---|
| `--blue` | `#3182f6` | 주요 액션, 강조 |
| `--blue-50` | `#e8f3ff` | 강조 배경 (선택 상태 등) |
| `--ink` | `#191f28` | 본문 텍스트 |
| `--ink-2` | `#4e5968` | 보조 텍스트 |
| `--ink-3` | `#8b95a1` | 약화 텍스트 |
| `--ink-4` | `#b0b8c1` | 비활성 |
| `--line` | `#e5e8eb` | 외곽선 |
| `--bg` | `#f9fafb` | 페이지 배경 |
| `--red` | `#f04452` | 경고·삭제 |
| `--green` | `#1eb18d` | 성공·순매출 양수 |
| `--amber` | `#f59e0b` | 주의 |

## 10. 반응형

### 10.1 브레이크포인트

| 너비 | 레이아웃 |
|---|---|
| ≥ 1280px | 데스크탑 풀 (사이드바 + 본문) |
| 1024 ~ 1279px | 데스크탑 축소 (사이드바 너비 축소) |
| 768 ~ 1023px | 태블릿 (사이드바 아이콘만) |
| < 768px | 모바일 (사이드바 햄버거 토글, 표는 가로 스크롤) |

### 10.2 표 처리

- 데스크탑 : 표 풀폭
- 모바일 : 가로 스크롤 + 좌우 스크롤 힌트 텍스트 "↔️ 좌우로 스크롤해서 모든 정보를 확인할 수 있어요"
- 핵심 컬럼은 sticky (예: 메뉴명)

---

## 11. 로깅 / 분석 (개발팀 참고)

- 사용자 액션 이벤트 추적 : 화면 진입, 주요 액션 클릭, 폼 제출
- 사장님 매장 ID, 사용자 역할(owner/staff), 화면명, 액션명을 함께 기록
- 에러 발생 시 스택 트레이스 + 매장 ID + 사용자 ID + 발생 시각 로깅
- 분석 도구 : 본사 선정 (예: Amplitude / Mixpanel / 자체 BI)
